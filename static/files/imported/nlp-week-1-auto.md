## 1. 什么是自然语言？

自然语言（Natural Language）是人类用来交流信息、思想与知识的工具。它和编程语言有本质区别。

自然语言具有几个典型特点：

- **持续演化**：新的词汇、表达、用法会不断出现。
- **不规则性**：语言中存在大量例外、习惯表达和非严格规则。
- **歧义性**：同一个词、句子甚至篇章，在不同上下文中可能对应不同含义。

因此，自然语言处理并不是简单地把语言看成一套固定规则，而是要研究：机器如何理解、生成、推理并与人类通过自然语言交互。

---

## 2. 什么是自然语言处理（NLP）？

自然语言处理（Natural Language Processing, NLP）是人工智能的一个分支，研究机器如何：

- 建模自然语言；
- 生成自然语言；
- 基于自然语言进行推理；
- 通过自然语言与人类交互。

NLP 的思想历史可以追溯到 20 世纪中期。slides 中提到的一个经典起点是 **Turing Test**：如果一个人通过自然语言和“人类”或“机器”对话，却无法可靠分辨哪一方是机器，那么机器就通过了测试。

这件事的重要意义不在于“考试本身”，而在于它揭示了一个核心问题：

> 如果机器能够在语言层面表现得像人一样，它是否就具备了某种智能？

因此，语言一直是 AI 的核心入口之一。

## 3. NLP 的发展历程

slides 将 NLP 的历史大致划分为三个阶段。

### 3.1 符号主义 NLP（1950s – early 1990s）

这一阶段的方法主要依赖人工编写规则。研究者会显式设计语法规则、词法规则、转换规则等，让系统完成翻译、分析或问答任务。

特点：

- 强依赖专家知识；
- 可解释性相对强；
- 迁移性与鲁棒性较差；
- 面对大规模真实语言现象时很难扩展。

### 3.2 统计 NLP（1990s – 2017）

随着计算能力提升和数据积累，机器学习方法开始取代大量手工规则。系统不再完全依靠人工指定规则，而是通过数据学习参数。

特点：

- 典型任务往往对应一个特定模型；
- 需要任务专用数据集；
- 从规则驱动转向数据驱动；
- HMM、CRF、n-gram、统计机器翻译等属于这一思路的代表。

### 3.3 大语言模型时代（2017 – 至今）

这一阶段最核心的变化是：

- 模型规模大幅扩大；
- 训练数据从“任务专用”转向“通用海量文本”；
- 任务形式从“每个任务一个模型”转向“统一的 text-to-text 范式”。

LLM 不再只为某一个任务单独训练，而是先在大规模通用文本上预训练，再通过指令微调、偏好对齐、强化学习等方式获得通用能力。这样，同一个模型可以完成问答、翻译、摘要、推理、对话、代码生成等多类任务。

## 4. LLM 发展中的关键节点

slides 提到了一些 LLM 历史上的代表性节点，可以把它们理解为“技术路线的里程碑”。

### 4.1 Transformer（2017）

Transformer 的提出奠定了现代 LLM 的核心结构基础。它最重要的意义在于：

- 用注意力机制替代传统循环结构；
- 更适合并行训练；
- 更容易扩展模型规模与上下文建模能力。

现代主流大模型几乎都建立在 Transformer 或其变体之上。

### 4.2 GPT-1（2018）与 BERT（2018）

这两个模型分别代表了两条重要路线：

- **GPT 系列**：偏向自回归生成；
- **BERT 系列**：偏向双向表示学习。

它们共同推动了“预训练 + 下游任务适配”的范式成为主流。

### 4.3 Scaling Law（2020）

OpenAI 对 scaling law 的系统研究表明：随着参数量、数据量和计算量增加，语言模型性能会以某种可预测方式提升。

这一发现的意义非常大，因为它说明：

> 很多能力并不一定来自精巧的人工设计，而可能来自更大规模的训练。

### 4.4 ChatGPT（2022）与 GPT-4（2023）

ChatGPT 让 LLM 首次以面向大众的对话产品形式被广泛使用。GPT-4 则进一步强化了多任务、推理、对齐和泛化能力，使 LLM 成为社会级应用基础设施。

slides 还继续列举了之后的一些节点，说明课程叙述的时间背景已经进入较新的 LLM 时代。其核心想表达的是：

- LLM 发展非常快；
- 能力提升不只体现在 benchmark 上；
- 也体现在 reasoning、tool use、multimodality、long context 等方面。

## 5. 传统 NLP 任务与现代 NLP 任务

slides 区分了“传统 NLP 任务”和“现代 NLP 任务”。

### 5.1 传统 NLP 任务

传统 NLP 任务通常是高复杂系统中的某一个子模块，例如：

- OCR
- 语音识别
- 分词
- 词性标注
- 句子切分
- 命名实体识别（NER）
- 实体链接
- 指代消解
- 词干提取
- 关系抽取等

这些任务有几个典型特点：

- 任务边界清晰；
- 模型和标签定义通常较固定；
- 常被串联成 pipeline；
- 很多早期方法强依赖规则与特征工程。

### 5.2 现代 NLP 任务

现代 NLP 更关注直接面向应用和智能行为的高层任务，例如：

- 问答
- 摘要
- 翻译
- 推理
- 多跳推理
- agent 行为
- continual learning
- watermarking
- jailbreak 分析等

这些任务往往：

- 更开放；
- 更难严格边界化；
- 不一定能被拆成固定 pipeline；
- 更需要模型具备通用语言理解与生成能力。

## 6. NLP 任务的统一：从 pipeline 到 text-to-text

这部分是 slides 很重要的一条主线。

在 LLM 之前，NLP 常常采用 pipeline 方式：

> 输入 → 分词 / 词性标注 / 依存分析 / NER / …… → 输出

也就是说：

- 每个任务都对应一个单独模型；
- 系统由多个模块串联而成；
- 前一模块的错误会向后传播。

而在 LLM 时代，越来越多任务被统一重写为：

> **text-to-text**

即：

- 输入是文本；
- 输出也是文本；
- 用户只需通过自然语言描述任务；
- 模型通过统一接口完成不同任务。

例如：

- 翻译：把中文翻成英文；
- 摘要：请总结下面文章；
- 信息抽取：请找出文中所有实体及关系；
- 推理：请一步步解释为什么答案是……

这意味着：

> “任务形式”本身被统一了。

这可以看作 LLM 对 NLP 的一个范式级重构。

## 7. 什么是大语言模型（LLM）？

slides 对 LLM 的定义比较清晰：

- 它也常被称为 foundation model / frontier model；
- 通常有极大的参数规模；
- 在海量无标注文本上通过 next-token prediction 进行预训练；
- 然后再通过 post-training 获得指令跟随、对齐、推理等能力。

### 7.1 预训练（Pre-training）

预训练的核心目标通常是 **next token prediction**：

给定前文，让模型预测下一个 token。

虽然这个目标看起来简单，但它迫使模型在大规模文本中学习：

- 词法与语法模式；
- 常识知识；
- 事实性知识；
- 一定程度的推理模式；
- 风格、体裁和上下文关联。

### 7.2 后训练（Post-training）

预训练后的模型通常还不够适合直接面向用户，因此需要 post-training。其作用包括：

- 学会遵循指令；
- 学会对话；
- 更符合人类偏好；
- 增强 reasoning；
- 降低不安全输出。

因此，从今天的角度看，一个真正可用的 LLM 往往不是“一个模型 + 一次训练”，而是一整条多阶段训练流水线。

## 8. The Bitter Lesson 与 LLM 的规模逻辑

slides 引用了 Rich Sutton 的 **The Bitter Lesson**。它的核心观点是：

> 从 AI 的长期发展看，真正最终胜出的往往是那些能够充分利用计算规模的通用方法，而不是人工嵌入大量特殊结构的方法。

放到 NLP/LLM 的语境里，这句话的意思是：

- 许多过去依赖人工设计的模块和规则，可能最终会被大规模训练的通用模型替代；
- “规模”本身是一条非常强的能力增长路线；
- 随着算力、数据和训练方法提升，模型能自动学习到越来越多的结构。

这也是为什么现代 LLM 研究如此强调：

- 参数规模；
- 数据规模；
- 训练算力；
- 推理阶段额外计算。

## 9. Scaling Laws 的意义

slides 中多次强调 scaling law，其想传达的不是一个公式细节，而是一种研究观念：

> 模型性能会随着规模增长而呈现某种可预测提升。

这件事带来的影响包括：

### 9.1 研究范式变化

过去很多 NLP 研究更关注：

- 特定结构设计；
- 更复杂的特征；
- 更强的任务特化。

而 scaling law 提示大家：

- 有时扩大规模比精细结构更有效；
- 通用训练范式可能比任务专用设计更有长期价值。

### 9.2 工业竞争方式变化

一旦 scaling law 成立，模型研发就不只是算法问题，也变成：

- 数据采集与清洗能力；
- 训练系统能力；
- 集群调度与工程优化能力；
- 推理成本控制能力。

因此，LLM 研究天然是算法、系统和产品三者交叉的方向。

## 10. 为什么 NLP 很重要？

slides 这一部分并不只是“举应用例子”，而是在说明：语言技术已经从研究问题变成社会基础设施。

原因至少有三层。

### 10.1 语言是人机交互最自然的接口

人类最熟悉的沟通方式就是自然语言。因此，谁掌握了语言接口，谁就更接近通用 AI 应用入口。

### 10.2 NLP 已经嵌入大量产品场景

slides 举到的例子包括：

- 搜索
- AI 眼镜
- 视频生成
- 输入法/语音输入
- Coding assistant

这些例子共同说明：

> NLP 已经不再局限于“文本处理”，而是成为多模态智能系统的核心中枢之一。

### 10.3 NLP 带来社会级影响

slides 提到：AI 正在影响内容生成、就业结构和人类决策方式。与此同时，也会带来风险，例如：

- 错误信息传播；
- 心理依赖；
- 对现实判断的扰动；
- 对劳动结构与知识生产方式的改变。

所以学习 NLP 不能只看能力边界，也要看到社会影响。

## 11. 经典 NLP 挑战

slides 提到几个经典但至今仍重要的 NLP 挑战。

### 11.1 Search Space 与 Recursion

自然语言具有递归性和组合性，这意味着：

- 句子结构可能非常复杂；
- 可能存在深层嵌套；
- 可表达空间巨大。

因此，语言理解不是简单的模板匹配，而是一个巨大搜索空间中的结构与语义建模问题。

### 11.2 概念漂移（Concept Drift）

语言不是静态的。词义会变化，新词会产生，旧表达会获得新含义。

例如：

- 某个词在 10 年前与今天的常见意义不同；
- 新出现的人名、产品、事件和网络用语不在旧语料里。

这意味着语言模型面临一个根本问题：

> 训练数据来自过去，但语言和世界在不断变化。

### 11.3 歧义（Ambiguity）

歧义是自然语言最核心的挑战之一，可能出现在：

- 词级别；
- 句子级别；
- 篇章级别。

slides 用 “bank” 这类词义歧义作为示例。其关键点是：

- 语言单位本身可能不唯一对应意义；
- 需要依赖上下文与背景知识消歧；
- LLM 的一个强项就是利用大规模上下文和预训练知识进行消歧。

## 12. 机器翻译：NLP 演进的一个缩影

slides 用机器翻译展示 NLP 方法演进路线：

- 规则方法；
- 统计方法；
- 神经方法；
- LLM 方法。

机器翻译之所以适合作为例子，是因为它完整体现了 NLP 的范式变迁。

### 12.1 规则时代

翻译依赖人工编写双语规则和词典。

### 12.2 统计时代

翻译转向通过平行语料学习词和短语的对应概率。

### 12.3 神经时代

翻译开始变成端到端序列建模问题。

### 12.4 LLM 时代

翻译不一定需要专门系统，而可以通过对话式 prompt 直接完成。

slides 同时指出：高资源语言之间的文本翻译已接近“基本解决”，但仍存在未解决问题，例如：

- 低资源语言；
- 文化细微差异；
- 语音到语音翻译；
- 复杂语境下的意图和风格保持。

这说明：一个任务“看起来很成熟”，不等于它所有层面都已被彻底解决。

## 13. LLM 时代的关键开放问题

slides 给出了当前 NLP/LLM 领域仍未解决的一批核心问题。这部分很重要，因为它告诉你：现代 NLP 的研究前沿在哪里。

### 13.1 World Modeling

模型是否真的形成了对世界的稳定内部表征？它的知识是“统计关联”还是“可操作的世界模型”？

### 13.2 Continual Learning

模型学到的新知识如何持续纳入，而不必总是大规模重训？

### 13.3 Long-Context Modeling

模型如何稳定利用极长上下文，而不仅仅是“窗口变大”？关键问题包括：

- 是否真正记住远处信息；
- 是否能高效检索重要信息；
- 上下文越长时推理是否仍可靠。

### 13.4 Tool Use

模型如何调用外部工具，例如：

- 搜索引擎
- 代码执行器
- API
- 数据库
- 浏览器

这让模型从“只会说”变成“能完成任务”。

### 13.5 Hallucination

模型会生成看似合理但实际上错误的内容。这是生成式模型最核心的可信度问题之一。

### 13.6 Reasoning

模型能否进行真实可靠的多步推理？它的 reasoning 是表面模仿，还是具有一定机制上的稳定性？

### 13.7 Efficiency

如何降低训练和推理成本，使更强模型变得更可部署、更可持续？

### 13.8 Alignment

如何让模型行为符合人类目标、价值与安全边界？

### 13.9 Persistent Memory / Interpretability

模型如何具有更长期、可管理的记忆？我们又如何理解模型内部到底学到了什么？

这些问题说明：今天的 NLP 并没有“做完”，只是研究重心已经从传统任务准确率，转向更高层次的智能能力与可靠性。

## 14. Continual Learning：为什么 LLM 很难“更新知识”？

slides 特别单独讲了 continual learning。核心问题是：

- LLM 的知识主要来自预训练阶段；
- 这些知识被分散编码在参数中；
- 一旦训练结束，想局部更新某类知识并不容易。

例如：如果模型训练结束后，世界上又出现了新的电影、事件、产品、术语，那么模型本身并不会自动知道这些知识。

这就引出两个方向：

1. **直接更新模型参数**；
2. **不改模型参数，而通过检索等外部机制补充新知识**。

因此 continual learning 和 retrieval、memory、editing 等方向彼此关系非常紧密。

## 15. 结构化知识、信息检索与 RAG

slides 在这部分的逻辑链条很清楚：

### 15.1 结构化知识

一个事实可以表示成三元组：

> (subject, relation, object)

例如：

> (William Shakespeare, write, Romeo and Juliet)

把知识结构化后，会更容易检索、更新与操纵。

### 15.2 信息检索（IR）

如果知识已经存储在数据库、索引或知识库中，那么系统可以按需检索相关内容，而不必完全依赖模型参数“记住一切”。

### 15.3 Retrieval-Augmented Generation（RAG）

RAG 的基本思想是：

1. 先根据问题去外部知识源中检索信息；
2. 把检索结果作为上下文提供给 LLM；
3. 再由 LLM 基于这些证据生成答案。

RAG 的主要意义：

- 缓解 hallucination；
- 提高事实可验证性；
- 让模型可以利用训练后才出现的新知识；
- 把“知识更新”从重新训练模型转移到更新知识库。

因此，RAG 可以被看作 LLM 与传统 IR/知识系统结合的一个典型范式。

## 16. Long Context：上下文变长不等于真的会用

slides 提到 long-context modeling，这是现代 LLM 中一个非常重要但容易被误解的话题。

“支持百万 token 上下文”并不自动意味着模型真正解决了长上下文问题。真正的问题包括：

- 模型能否记住前面很远处的重要信息；
- 能否在长上下文中找到与当前问题最相关的片段；
- 能否跨越长距离进行一致推理；
- 长上下文是否会带来噪声干扰和注意力分散。

因此，long-context 不只是“窗口大小”问题，更是：

> **检索、压缩、选择、记忆与推理的联合问题。**

## 17. Test-Time Scaling 与 Chain-of-Thought

slides 中另一条关键主线是：**test-time scaling**。

它的核心思想是：

> 模型在推理时多花一些计算与时间，可能得到更好的答案。

### 17.1 Chain-of-Thought（CoT）

CoT 指的是让模型显式地产生中间推理步骤，而不是直接给最终答案。

例如原本只问“答案是什么”，现在改成：

- 请一步一步分析；
- 请展示中间推理过程；
- 请先拆解问题再得出结论。

这样做的好处是：

- 复杂问题被分解为多个子步骤；
- 模型有更多机会自我修正；
- 某些推理任务性能显著提升。

### 17.2 Test-Time Scaling 的本质

过去 scaling 更多指训练时扩大模型和数据；而这里强调的是：

- 在测试时也可以增加“思考计算量”；
- 让模型花更多 token 做规划、推理、验证；
- 从而提升最终表现。

这使得“推理能力”不再仅仅依赖参数规模，也和测试时资源分配密切相关。

## 18. Reasoning Models

slides 把 reasoning model 看成 LLM 的一个重要分支。它们的目标不是只做 fluent generation，而是更擅长：

- 多步逻辑推理；
- 数学与代码问题求解；
- 复杂决策与规划。

其中一个重要训练路线是强化学习（RL）：

- 模型探索不同推理路径；
- 外部奖励信号评估这些路径质量；
- 模型逐渐学会更有效的推理方式。

这一趋势说明：现代 LLM 研究已经从“会说话”逐渐走向“会思考、会行动、会使用工具”。

## 19. Hallucination：LLM 的核心可靠性问题

Hallucination 指模型生成了错误甚至虚假的内容，却以事实口吻表达出来。

它为什么危险？因为它通常不是“胡言乱语式错误”，而是：

- 表面上看起来流畅、专业、可信；
- 但底层事实不成立；
- 用户不容易第一时间察觉。

Hallucination 的来源可能包括：

- 训练数据噪声；
- 模型倾向于生成“最像正确答案”的文本；
- 缺乏真实外部验证；
- 推理链条中前面某一步错误被后续放大。

RAG、tool use、calibration、verification 等方法，很多都是在试图降低 hallucination。

## 20. Alignment：能力之外还要“行为正确”

Alignment 关注的问题是：

> 模型即使能力很强，它的目标和行为是否真的符合人类意图？

这不是简单的“让模型更礼貌”，而是涉及：

- 是否遵循人类偏好；
- 是否避免危险输出；
- 是否在多目标场景下做出合理权衡；
- 是否会在复杂环境中偏离预期目标。

因此，alignment 是现代 LLM 研究中的核心安全主题之一。

## 21. NLP 的应用已经走向多模态与 agent 化

slides 通过很多应用例子说明：NLP 已经不再只是传统文本任务。

它正朝两个方向扩张：

### 21.1 多模态化（Multimodality）

语言模型开始结合：

- 图像
- 音频
- 视频
- 语音

语言成为统一的控制与解释接口，而感知信息来自多个模态。

### 21.2 Agent 化（Agentic Behavior）

模型不再只是回答问题，而是可以：

- 调工具；
- 执行 API；
- 浏览网页；
- 分解多步任务；
- 自主规划并完成目标。

这意味着 NLP 的边界正在和智能体系统、软件系统、操作系统接口、真实世界任务执行逐渐融合。

## 22. 近期 NLP 研究趋势

slides 最后总结了一些当前趋势，可以视作你读近期论文时的重要导航。

### 22.1 Scaling Laws

继续扩大模型、数据和训练规模。

### 22.2 Densing Laws

在达到同等性能的前提下，模型所需参数规模可能以可预测方式下降，说明架构和训练方式也在进步。

### 22.3 Towards Multimodality

前沿模型越来越不再局限于文本，而是统一处理视觉、音频等信息。

### 22.4 Scaling Test-Time Compute

通过在推理阶段投入更多计算，增强 reasoning 与复杂任务求解能力。

### 22.5 Towards Infinite Context Length

模型上下文窗口持续增大，但真正挑战在于如何有效利用这些上下文。

### 22.6 Agentic Behavior and Tool Use

模型与外部工具、环境和程序接口结合，向“可执行智能系统”发展。

这些趋势共同说明：

> 现代 NLP 已经从“文本分析领域”变成“通用智能接口与语言驱动系统”的核心组成部分。

## 23. 总结：这节 introduction 真正想建立什么框架？

如果把整套 slides 的知识点压缩成一个总框架，可以概括为下面几层。

### 23.1 第一层：NLP 的对象

NLP 研究的是自然语言，而自然语言具有演化性、不规则性和歧义性，因此远比形式化语言复杂。

### 23.2 第二层：NLP 的历史演进

NLP 经历了：

- 符号主义；
- 统计学习；
- 深度学习与 LLM 统一范式。

### 23.3 第三层：LLM 带来的范式变化

LLM 让大量任务统一成 text-to-text，并把预训练 + 后训练变成核心方法论。

### 23.4 第四层：NLP 的现实重要性

语言技术已经成为 AI 产品、社会系统与人机交互的关键基础设施。

### 23.5 第五层：当前研究前沿

即使传统任务已有大量突破，现代 NLP 仍面临一系列关键开放问题：

- 持续学习
- 长上下文
- 工具使用
- 推理
- 幻觉
- 对齐
- 效率
- 可解释性
- 持久记忆

因此，这节课表面上是 introduction，实际上在做两件事：

1. 帮你建立 NLP/LLM 的整体地图；
2. 帮你理解今天最值得研究的问题在哪里。
