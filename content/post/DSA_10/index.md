# 数据结构与算法笔记（十）：图

> 本文基于课程第十章“图”整理。内容不只是按 PPT 顺序摘录，而是在保留课程主线的基础上，尽量把“图的基本概念、邻接矩阵与邻接表、BFS、DFS、边分类、最短路径直觉、拓扑排序”等内容串成一篇适合直接发布到博客的 Markdown 笔记。

## 1. 为什么树之后一定要学图

### 1.1 树是特殊的，图才是更一般的关系模型

在前面的章节中，我们已经学过：

- 向量、列表：线性关系
- 栈、队列：受限线性关系
- 树、二叉树、搜索树：层次关系

但这些结构都有明显限制：

- 列表中一个节点通常只与前后相邻元素相关
- 树中每个节点只有一个父亲
- 二叉树中节点的分支数甚至进一步受限

现实世界里的关系却经常远比这些复杂。  
例如：

- 城市之间的交通网络
- 网页之间的超链接
- 社交网络中的好友关系
- 课程之间的先修关系
- 任务之间的依赖关系

这些对象之间常常体现为“任意多对多”的连接，而不再是简单的层级或线性关系。

因此：

> **图（graph）是比树更一般的结构。**

树可以看作一种特殊图：

- 连通
- 无环
- 边数恰好为 `n-1`

而图则允许：

- 更一般的连接方式
- 环
- 多条路径
- 局部稠密、整体稀疏等复杂形态

---

## 1.2 图论的核心任务

一旦进入图，我们要处理的典型问题就会变成：

- 哪些顶点彼此连通？
- 从一个点能到达哪些点？
- 最短路是什么？
- 图中是否有环？
- 能否把依赖关系排成一个线性顺序？
- 某条边在遍历过程中属于什么角色？
- 这张图适合用什么方式存储？

因此这一章虽然标题只是“图”，但真正的学习重点有两层：

1. **如何表示图**
2. **如何遍历图**

而后者又几乎是一切图算法的基础。

---

## 2. 图的基本概念

## 2.1 图的定义

课件第 2 页给出图的一般记法：

$
G = (V, E)
$

其中：

- `V`：顶点集合（vertex set）
- `E`：边集合（edge / arc set）

并记：

- `n = |V|`：顶点数
- `e = |E|`：边数

这是图最基本的两个规模参数。

---

### 2.2 邻接（adjacency）与关联（incidence）

课件第 2 页给出两个最基本术语：

#### 2.2.1 邻接（adjacency）

若一条边连接顶点 `u` 和 `v`，则说：

- `u` 与 `v` 邻接
- `v` 与 `u` 邻接

邻接描述的是**顶点与顶点之间**的直接关系。

#### 2.2.2 关联（incidence）

若顶点 `v` 是某条边 `e` 的一个端点，则说：

- 顶点 `v` 与边 `e` 关联

关联描述的是**顶点与边之间**的从属关系。

这两个概念很容易混，但它们分别对应：

- adjacency：顶点-顶点
- incidence：顶点-边

---

### 2.3 自环与简单图

课件第 2 页指出：

- 若同一顶点与自己邻接，则形成**自环（self-loop）**
- 若图中既没有自环，也没有重边（多条重复边），则称为**简单图（simple graph）**

本章主要讨论简单图。  
这是为了把注意力集中在：

- 图的存储
- 图的遍历
- 图的算法逻辑

而不被特殊边界情况干扰。

---

### 2.4 度（degree / valency）

课件第 2 页给出：

> 与同一顶点关联的边数，称为该顶点的度（degree）。 

在无向图中：

- 度就是与该顶点接触的边数

在有向图中，还要进一步区分：

- **入度（in-degree）**
- **出度（out-degree）**

这个区分在后面拓扑排序、可达性分析中尤其重要。

---

## 3. 无向图、有向图与混合图

## 3.1 无向边与无向图

课件第 3 页指出：

若邻接顶点 `u` 和 `v` 的次序无关，则：

$
(u, v)
$

是一条**无向边（undirected edge）**。

若一张图中所有边都无方向，则它是：

> **无向图（undigraph）**

无向图适合表示：

- 对称关系
- 双向交通
- 无方向的连接
- “是否相连”这种纯连接信息

---

## 3.2 有向边与有向图

课件第 3 页进一步指出：

若边有方向，则称为**有向边（directed edge）**，也常称作弧（arc）。

对于有向边 `(u, v)`：

- `u` 叫尾（tail）
- `v` 叫头（head）

若图中所有边都有方向，则称为：

> **有向图（digraph）**

有向图能表达更丰富的关系，例如：

- 单向依赖
- 单向到达
- 先修课程关系
- 任务先后约束
- 网页链接方向

课件也明确说：

> 有向图通用性更强，因此本章主要针对有向图介绍相关结构及算法。 

---

## 3.3 混合图

若一张图中：

- 无向边与有向边并存

则称为：

> **混合图（mixed graph）** 

这类图在理论上也常见，但本章主要聚焦无向图和有向图。

---

## 4. 路径、环路与 DAG

## 4.1 路径（path）

课件第 4 页定义路径：

$
\pi = < v_0, v_1, ..., v_k >
$

若相邻顶点之间均有边相连，则称其为一条路径。  
路径长度定义为：

$
|\pi| = k
$

也就是边数，而不是顶点数。

---

## 4.2 简单路径（simple path）

若路径中没有重复顶点，即：

$
v_i 
e v_j \quad (i 
e j)
$

则称之为：

> **简单路径（simple path）** 

简单路径的意义在于：

- 它不绕圈
- 不重复访问已走过的顶点
- 往往更能体现“纯粹的到达关系”

---

## 4.3 环 / 环路（cycle）

若路径满足：

$
v_0 = v_k
$

则形成一个环（cycle）。  
若除起点与终点重合外其余顶点不重复，则是简单环。

环的存在会直接影响很多图算法问题，例如：

- 是否可做拓扑排序
- DFS 中会不会出现后向边
- 图是否为 DAG
- 是否存在依赖回路

---

## 4.4 DAG：有向无环图

课件第 4 页给出：

> **有向无环图（DAG, Directed Acyclic Graph）** 

这类图非常重要，因为很多“依赖关系”天然对应 DAG，例如：

- 课程先修
- 编译依赖
- 项目流程
- 任务调度
- 知识点先后

只要图中一旦有环，就意味着：

- 存在循环依赖
- 不可能给出完全符合依赖关系的线性执行顺序

这也是后面拓扑排序要解决的问题。

---

## 4.5 欧拉回路与哈密尔顿回路

课件第 4 页还列出两类经典特殊回路：

### 4.5.1 欧拉回路（Eulerian tour）

要求：

- 所有边恰好经过一次
- 因此路径长度满足：

$
|\pi| = |E|
$

### 4.5.2 哈密尔顿回路（Hamiltonian tour）

要求：

- 所有顶点恰好经过一次
- 因此路径长度满足：

$
|\pi| = |V|
$

这两个概念虽然本章不深入展开算法，但它们说明：

> 图上的“遍历对象”有时是边，有时是顶点，问题类型差别极大。

---

## 5. 支撑树、带权网络与最小支撑树

## 5.1 支撑树（spanning tree）

课件第 5 页指出：

若图 `G = (V, E)` 的某个子图：

$
T = (V, F)
$

满足：

- 仍包含全部顶点 `V`
- 且本身是一棵树

则称 `T` 为 `G` 的：

> **支撑树（spanning tree）**

支撑树保留了图的“连通骨架”，但去除了所有多余回路。

---

## 5.2 带权网络（weighted network）

若每条边 `e` 都有一个权值：

$
wt(e)
$

则图称为：

> **带权网络（weighted network）** 

这时图不再只是“连没连”，而还带有：

- 长度
- 费用
- 时间
- 带宽
- 风险
- 概率
- 相似度

等信息。

---

## 5.3 最小支撑树（MST）

同一网络的所有支撑树中，总权重最小者称为：

> **最小支撑树（Minimum Spanning Tree, MST）** 

虽然这一章主要聚焦遍历和表示，还没展开 Prim / Kruskal 等算法，但课件在开头先提到 MST，是为了说明：

> 图的价值不只在“遍历”，还在“优化问题”。

---

## 6. 图的模板抽象：统一接口比实现更重要

## 6.1 Graph 抽象模板类

课件第 7 页给出统一图模板类：

```cpp
template <typename Tv, typename Te> class Graph {
private:
    void reset() { /* 所有顶点、边的辅助信息复位 */ }
public:
    int n, e; // 顶点数、边数
    /* ... 顶点操作、边操作、图算法：无论如何实现，接口必须统一 ... */
};
```

这里非常重要的一点是：

> **无论图是用邻接矩阵实现，还是邻接表实现，算法接口必须统一。**

这和前面 ADT 思想一脉相承：

- 接口定义“做什么”
- 具体存储决定“怎么做”

于是：

- BFS、DFS、拓扑排序等算法
- 应该写在统一抽象接口之上
- 而不被某种具体实现绑死

---

## 6.2 `reset()` 的意义

课件第 7 页的 `reset()` 会为所有顶点与边重置辅助信息：

- 顶点：
  - `status = UNDISCOVERED`
  - `dTime = -1`
  - `fTime = -1`
  - `parent = -1`
  - `priority = INT_MAX`
- 边：
  - `type = UNDETERMINED`

这说明：

> 图遍历算法通常并不只“访问一下节点”，  
> 它们还需要在运行过程中记录丰富的辅助标签。

比如：

- 是否已发现
- 发现时间 / 访问完成时间
- 在遍历树中的父节点
- 边属于 TREE / CROSS / BACKWARD / FORWARD 哪类

这也是图算法相较于树遍历更复杂的原因之一。

---

## 7. 邻接矩阵：最直观的图表示

## 7.1 定义

课件第 8 页给出邻接矩阵（adjacency matrix）：

用一个 `n × n` 矩阵 `A` 来记录顶点之间的邻接关系。

最简单形式下：

- `A(i,j) = 1`：若存在边 `(i,j)`
- `A(i,j) = 0`：否则

对于简单图：

- 对角线统一为 `0`
- 因为不考虑自环

---

## 7.2 邻接矩阵的空间复杂度

无论图中实际有多少边，邻接矩阵都必须存满整个 `n × n` 区域。  
因此空间复杂度恒为：

$
\Theta(n^2)
$

这是它最大的特点，也是最大的代价。课件第 8 页明确强调：空间与实际边数无关。 

---

## 7.3 关联矩阵（incidence matrix）

课件第 8 页还提到另一种矩阵表示：关联矩阵。

它记录的是：

- 顶点与边之间的关联关系

其空间复杂度是：

$
\Theta(n \cdot e) = O(n^3)
$

通常更大，空间利用率很低:

$2e / ne = 2 / n$。 

不过在某些特定问题中会很有效。

本章主要还是围绕邻接矩阵与邻接表。

---

## 7.4 邻接矩阵的示例

课件第 9 页给出了：

- 无向图
- 有向图
- 带权网络

三种图在邻接矩阵中的表示方式。

其中很容易看到：

- 无向图矩阵关于主对角线对称
- 有向图则不必对称
- 带权图中矩阵元素不再只是 `0/1`，而是边权或 `∞`

这说明邻接矩阵其实不仅能表示“有无边”，也能自然表示：

- 有向性
- 权重
- 边类型等附加信息

---

## 8. 邻接矩阵的模板实现

## 8.1 Vertex 结构

课件第 11 页给出顶点结构体：

```cpp
using VStatus = enum { UNDISCOVERED, DISCOVERED, VISITED };

template <typename Tv> struct Vertex {
    Tv data;
    int inDegree, outDegree;
    VStatus status;
    int dTime, fTime;
    Rank parent;
    int priority;
};
```

这里不仅保存顶点原始数据 `data`，还保存：

- 入度 / 出度
- 遍历状态
- 发现时间 / 完成时间
- 遍历树父节点
- 优先级

可以看到：

> 图节点不仅要“存值”，还要承担大量算法状态。

---

## 8.2 Edge 结构

课件第 12 页给出边结构体：

```cpp
using EType = enum { UNDETERMINED, TREE, CROSS, FORWARD, BACKWARD };

template <typename Te> struct Edge {
    Te data;
    int weight;
    EType type;
};
```

它包含：

- 边自身数据
- 权重
- 在遍历树中的边类型

这也是图不同于简单集合的地方：

> **边本身也可以是“有状态的对象”。**

---

## 8.3 GraphMatrix 实现

课件第 13 页给出邻接矩阵实现框架：

```cpp
template <typename Tv, typename Te>
class GraphMatrix : public Graph<Tv, Te> {
private:
    Vector< Vertex<Tv> > V;
    Vector< Vector< Edge<Te>* > > E;
};
```

也就是说：

- `V` 存顶点表
- `E[i][j]` 存边 `(i,j)` 的指针
- 若边不存在则为 `NULL`

这比最简单的 `0/1` 邻接矩阵更通用，因为它可以直接挂：

- 边对象
- 权重
- 类型
- 附加信息

---

## 9. 邻接矩阵上的静态操作

## 9.1 顶点读写

课件第 15 页列出一系列 O(1) 接口：

- `vertex(i)`
- `inDegree(i)`
- `outDegree(i)`
- `status(i)`
- `dTime(i)`
- `fTime(i)`
- `parent(i)`
- `priority(i)`

由于顶点都放在顺序表里，因此这些访问自然都是常数时间。

---

## 9.2 边读写与存在性判断

课件第 16 页给出：

```cpp
bool exists(i, j) { return (i < n) && (j < n) && E[i][j] != NULL; }
```

于是：

- 判断边是否存在：`O(1)`
- 访问 `edge(i,j)` / `type(i,j)` / `weight(i,j)`：也都是 `O(1)`

这正是邻接矩阵最大优势之一：

> **判断任意两点间是否有边，可以瞬时完成。**

---

## 9.3 邻接点枚举

课件第 17 页给出：

```cpp
Rank firstNbr(Rank i) { return nextNbr(i, n); }

Rank nextNbr(Rank i, Rank j) {
    while ((-1 < j) && !exists(i, --j));
    return j;
}
```

它的含义是：

- 若已枚举到邻居 `j`
- 就从 `j-1` 开始逆向继续扫描，找下一个存在边的位置

由于邻接矩阵必须扫整行，所以枚举一个顶点的所有邻接点需要：

$
O(n)
$

课件特别注明：

> 若改用邻接表，可提高到 `O(1 + outDegree(i))`。 

这也正是后面引入邻接表的原因。

---

## 10. 邻接矩阵上的动态操作

## 10.1 插入边

课件第 19 页：

```cpp
void insert(Te const& edge, int w, Rank i, Rank j) {
    if (exists(i, j)) return;
    E[i][j] = new Edge<Te>(edge, w);
    e++;
    V[i].outDegree++;
    V[j].inDegree++;
}
```

可见：

- 插入边本身是 `O(1)`
- 同时还可即时维护入度 / 出度

---

## 10.2 删除边

课件第 20 页：

```cpp
Te remove(Rank i, Rank j) {
    Te eBak = edge(i, j);
    delete E[i][j]; E[i][j] = NULL;
    e--;
    V[i].outDegree--;
    V[j].inDegree--;
    return eBak;
}
```

同样是 `O(1)`，并且度数同步更新。

---

## 10.3 插入顶点

课件第 21 页给出的思路是：

1. 先给已有每一行都加一个新列
2. 再增加一整行
3. 最后在 `V` 中加入新顶点

由于矩阵边长增加，插入顶点需要对所有已有行做修改，因此是：

$
O(n)
$

---

## 10.4 删除顶点

课件第 22 页给出删除逻辑：

1. 删除该顶点所有出边
2. 删除矩阵中的整行
3. 删除顶点对象
4. 再删除所有入边及对应整列

这也需要对矩阵做整行整列处理，因此成本显然不低，通常也是：

$
O(n^2)
$

量级上的操作。

这也说明：

> 邻接矩阵更适合“顶点集规模相对固定”的图。

---

## 11. 邻接矩阵的优缺点

## 11.1 优点

课件第 24 页总结了邻接矩阵的优势：

1. **直观，易于理解和实现**
2. **适用范围广**
3. 尤其适合**稠密图（dense graph）**
4. 判断两点间是否存在联边：`O(1)`
5. 获取入度 / 出度：`O(1)`
6. 添加 / 删除边后更新度数：`O(1)`

这些特点使得邻接矩阵在以下场景中非常强：

- 图很稠密
- 经常做 `exists(i,j)` 判断
- 顶点集基本固定
- 需要快速访问边权

---

## 11.2 缺点

课件第 25 页指出：

邻接矩阵最大缺点是空间：

$
\Theta(n^2)
$

对于稀疏图而言，这会严重浪费。  
课件还特别以平面图为例说明：

- 平面图满足：

$
e \le 3n - 6 = O(n)
$

于是若仍用 `n^2` 空间存储，利用率约为：

$
1/n
$

这随着 `n` 增大而迅速变得极低。

---

## 12. 邻接表：为稀疏图而生

## 12.1 基本思想

课件第 27 页给出邻接表的出发点：

> 将邻接矩阵的各行组织为列表，只记录存在的边。

也就是说，对每个顶点 `v`，维护其邻接顶点列表：

$
L_v = \{ u \mid < v,u > \in E \}
$

这就把原来稀疏矩阵中大量的 `0 / NULL` 单元全部省掉了。

---

## 12.2 邻接表的空间复杂度

课件第 29 页总结：

- 有向图：`O(n + e)`
- 无向图：`O(n + 2e)`，仍是 `O(n + e)` ***如何改进***

因为无向边会在两个顶点的表中各存一次。

这比邻接矩阵的 `Θ(n^2)` 对稀疏图来说好得多。

特别对于平面图： ***什么是平面图***

$
e = O(n)
$

于是总空间仅为：

$
O(n)
$

较邻接矩阵是极大改进。 

---

## 12.3 邻接表上的时间复杂度

课件第 30～31 页总结了一组非常重要的复杂度对比：

### 优势

- 枚举某顶点所有出边：`O(1 + deg(v))`
- 枚举无向图中某顶点所有邻居：`O(1 + deg(v))`
- 适合做遍历

### 劣势

- 判断给定 `(u,v)` 是否存在边：
  - 需搜索 `u` 的邻接表
  - 最坏是 `O(deg(u))`
  - 仍可能退化到 `O(e)`

这与邻接矩阵恰好形成互补：

- 邻接矩阵：边存在性判断快
- 邻接表：枚举邻居更快、空间更省

---

## 12.4 何时选邻接矩阵，何时选邻接表

课件第 32 页给出一张取舍表：

### 邻接矩阵适合

- 经常检测边是否存在
- 经常做边插入 / 删除
- 图规模固定
- 图较稠密

### 邻接表适合

- 经常计算顶点度数
- 顶点数目不确定
- 经常做遍历
- 图较稀疏

这张表很值得记住，因为它不是抽象理论，而是非常实用的工程经验。

# 第二部分：广度优先搜索 BFS

## 13. BFS 的直观思想

## 13.1 分层扩张

课件第 34 页给出 BFS 的文字定义：

从顶点 `s` 出发：

1. 访问 `s`
2. 依次访问 `s` 所有尚未访问的邻接顶点
3. 再依次访问它们的所有尚未访问邻接顶点
4. 如此反复，直到没有新的可访问顶点

这本质上就是：

> **按距离一层一层向外扩张。**

---

## 13.2 为什么 BFS 像树的层次遍历

课件第 34 页直接指出：

> BFS 的策略与过程，完全等同于树的层次遍历。

只不过图中因为可能有环和交叉连接，所以必须多做一件事：

- 记录某顶点是否已经被发现过

否则就会在环里无限打转。

---

## 13.3 BFS 生成 BFS 树

课件第 34 页还强调：

> BFS 会构造出原图的一棵支撑树（BFS tree）。 

含义是：

- 每当 BFS 首次发现一个新顶点 `u`
- 都是通过某个顶点 `v` 到达它
- 那么就把边 `(v,u)` 记作一条树边
- 所有这些树边最终连成一棵以 `s` 为根的 BFS 树

---

## 14. BFS 的算法实现

## 14.1 队列是核心

课件第 36～37 页给出标准 BFS：

```cpp
Queue<Rank> Q;
status(v) = DISCOVERED;
Q.enqueue(v);

while (!Q.empty()) {
    Rank v = Q.dequeue();
    dTime(v) = ++clock;
    for (Rank u = firstNbr(v); -1 < u; u = nextNbr(v, u))
        if (UNDISCOVERED == status(u)) {
            status(u) = DISCOVERED;
            Q.enqueue(u);
            type(v, u) = TREE;
            parent(u) = v;
        } else
            type(v, u) = CROSS;
    status(v) = VISITED;
}
```

队列恰好保证了：

- 先发现的顶点先被扩展
- 也就保证了“按层推进”

---

## 14.2 三种状态的含义

BFS 中顶点状态有三种：

- `UNDISCOVERED`：尚未发现
- `DISCOVERED`：已发现但尚未完全访问完
- `VISITED`：所有邻居都考察完毕

这比树遍历多出一层“已发现但未完成”的中间状态，原因就在于图中存在：

- 环
- 重复到达
- 交叉边

---

## 14.3 边分类：TREE 与 CROSS

课件第 37 页给出 BFS 中的边分类：

- 若 `u` 尚未发现，则 `(v,u)` 是 **TREE 边**
- 否则 `(v,u)` 是 **CROSS 边**

BFS 中边类型比较简单：

- 不像 DFS 那样有 TREE / BACKWARD / FORWARD / CROSS 四类
- 主要分为 TREE 与 CROSS

这是因为 BFS 按层推进，结构相对规整。

---

## 15. BFS 的推广：连通分量与可达分量

## 15.1 只遍历一个分量

课件第 42 页指出：

- 在无向图中，从 `s` 出发的 BFS 会访问 `s` 所在连通分量的全部顶点
- 在有向图中，会访问从 `s` 可达的全部顶点

因此若只关心：

- 某个顶点所在连通块
- 某个源点的可达区域

那么一次 BFS 就够了。

---

## 15.2 遍历整张图：`bfs(s)`

若图不连通（或有向图中存在多个不可达分量），一次 BFS 不够。  
课件第 43 页给出外层框架：

```cpp
reset();
int clock = 0;
Rank v = s;
do
    if (UNDISCOVERED == status(v))
        BFS(v, clock);
while (s != (v = ((v+1) % n)));
```

也就是：

- 从起点 `s` 开始，按编号循环检查所有顶点
- 一旦发现还有未发现顶点，就再从那里启动一次 BFS

最终会得到：

> **一个 BFS 森林（BFS forest）**

---

## 15.3 BFS 的总体复杂度

课件第 44 页分析：

若采用邻接表，则：

- 每个顶点入队、出队各一次
- 每条边被考察常数次

因此总时间复杂度为：

$
O(n + e)
$

这也是图遍历最经典、最重要的线性复杂度。

---

## 16. BFS 的性质与应用

## 16.1 BFS 树 / BFS 森林

课件第 47 页指出：

对于每个连通 / 可达分量：

- `BFS(v)` 恰好启动一次
- 它会生成一棵以 `v` 为根的 BFS 树

整体 `bfs()` 则生成一个 BFS 森林。  
森林中：

- 树边数为 `n - c`
- 其中 `c` 为分量数

---

## 16.2 BFS 队列像一条“贪吃蛇”

课件第 48 页用了一个非常形象的比喻：

BFS 队列 `Q` 像一条“贪吃蛇”：

- 队列中的顶点按 `dist(s)` 单调排列
- 相邻顶点距离之差不超过 1
- 树边连接的顶点距离恰差 1
- 跨边连接的顶点距离至多差 1

这其实揭示了 BFS 最核心的层次结构：

> **BFS 队列天然维持着从源点出发的距离波前。**

---

## 16.3 最短路径性质

课件第 48 页给出最重要应用结论：

> 在无权图中，BFS 树中从 `s` 到 `v` 的路径，就是原图中 `s` 到 `v` 的最短路径。

原因很简单：

- BFS 按层扩张
- 顶点第一次被发现时，已经是通过最少边数到达的

这使得 BFS 成为解决以下问题的天然工具：

- 无权最短路
- 最少步数问题
- 社交网络中的“几度关系”
- 图上的层次划分

课件第 49～50 页用 Erdős Number 和 Chow Number 举例，正是在说明这种“合作距离 / 关系距离”本质上就是 BFS 层数。

# 第三部分：深度优先搜索 DFS

## 17. DFS 的直观思想

## 17.1 “一条路走到黑，再回头”

课件第 52 页定义 DFS：

从顶点 `s` 出发：

1. 访问 `s`
2. 若 `s` 尚有未访问邻居，则任选其一 `u`
3. 递归执行 `DFS(u)`
4. 若没有未访问邻居，则回溯返回

这与 BFS 的“逐层扩张”完全不同。  
DFS 的策略更像：

> **先尽可能深入，再在走不下去时回头。**

---

## 17.2 为什么 DFS 像树的先序遍历

课件第 52 页指出：

对于树而言，DFS 等效于先序遍历。  
在图中，它同样会生成一棵 DFS 树，只是因为图里可能有额外边，所以需要更复杂的边分类。

---

## 18. DFS 的算法实现

## 18.1 递归框架

课件第 53～54 页给出标准 DFS：

```cpp
dTime(v) = ++clock;
status(v) = DISCOVERED;

for (Rank u = firstNbr(v); -1 < u; u = nextNbr(v, u))
    switch (status(u)) {
        case UNDISCOVERED:
            type(v, u) = TREE;
            parent(u) = v;
            DFS(u, clock);
            break;
        case DISCOVERED:
            type(v, u) = BACKWARD;
            break;
        default:
            type(v, u) = dTime(v) < dTime(u) ? FORWARD : CROSS;
            break;
    }

status(v) = VISITED;
fTime(v) = ++clock;
```

和 BFS 比较，DFS 最大特点是：

- 真正依赖递归
- 同时记录发现时间 `dTime`
- 以及访问完成时间 `fTime`

---

## 18.2 为什么 DFS 需要 `dTime` 与 `fTime`

在树中不太需要这两个时间戳。  
但在图中，DFS 要靠它们回答很多问题：

- 某个节点是不是另一个节点的祖先 / 后代
- 一条边是前向边、后向边还是跨边
- 图中是否有环
- 能否拓扑排序

所以：

> `dTime / fTime` 是 DFS 理解图结构的关键辅助信息。

---

## 19. DFS 的边分类

课件第 54 页与第 74 页给出了 DFS 的边分类：

### 19.1 TREE 边

若从当前顶点 `v` 进入一个尚未发现的顶点 `u`，则：

$
(v,u)
$

是 TREE 边。

这表示 DFS 树在此处真正向外扩展。

---

### 19.2 BACKWARD 边

若试图从当前顶点 `v` 进入一个仍处于 `DISCOVERED` 状态的顶点 `u`，则：

$
(v,u)
$

是 BACKWARD 边。

它意味着：

- `u` 是当前递归路径上的祖先
- 也因此说明图中存在回路

课件第 74 页明确写出：

> DFS 发现后向边 iff 存在回路

这条结论极其重要，因为很多图算法正是靠 DFS 的后向边来判环。

---

### 19.3 FORWARD 边

若 `u` 已经 `VISITED`，且：

$
dTime(v) < dTime(u)
$

则 `(v,u)` 是 FORWARD 边。

这说明：

- `u` 是 `v` 在 DFS 树中的某个后代
- 但并不是通过这条边第一次发现的

---

### 19.4 CROSS 边

若 `u` 已 `VISITED`，且：

$
dTime(u) < dTime(v)
$

则 `(v,u)` 是 CROSS 边。

它表示：

- `u` 与 `v` 不存在直接祖先-后代关系
- 它们可能属于不同分支，甚至不同 DFS 树

---

## 20. DFS 森林与括号引理

## 20.1 DFS 森林

与 BFS 类似，若图不连通，则需要外层包装：  
课件第 61～62 页给出 `dfs(s)`，会对所有尚未发现的顶点再启动一次 DFS。

于是最终得到：

> **DFS 森林（DFS forest）**

对于每个分量，会有一棵 DFS 树。

---

## 20.2 Parenthesis Lemma（括号引理）

课件第 73 页给出 DFS 最经典的性质之一：

把每个顶点 `u` 的活跃期记作：

$
active[u] = (dTime[u], fTime[u])
$

则有：

- `u` 是 `v` 的后代  
  iff `active[u] ⊆ active[v]`
- `u` 是 `v` 的祖先  
  iff `active[u] ⊇ active[v]`
- `u` 与 `v` 无祖先后代关系  
  iff `active[u] ∩ active[v] = ∅`

这称为：

> **括号引理（Parenthesis Lemma）**

它说明 DFS 的时间区间像一串正确嵌套的括号。  
这个视角极其重要，因为很多判定都能用时间戳直接完成，而无需真的再走树。

---

## 21. DFS 的性质与应用

课件第 72～75 页以及 75 页最后一张总结页列出了一系列 DFS / BFS 的应用。

### 21.1 DFS 能解决什么

- 连通性检测
- 可达性检测
- 环检测
- 拓扑排序
- 双连通分量
- 强连通分量
- 欧拉回路（配合更专门逻辑）
- 许多图结构性质分析

### 21.2 BFS 更擅长什么

- 无权最短路径
- 距离层次
- 图半径 / 直径 / 围长 / 中心等问题
- 连通性 / 可达性（当然也可用 DFS）

因此可以粗略记忆为：

> **BFS 更偏“距离 / 层次”**  
> **DFS 更偏“结构 / 依赖 / 回路 / 递归分解”**

# 第四部分：拓扑排序

## 22. 为什么 DAG 需要拓扑排序

## 22.1 DAG 的典型含义

课件第 77 页给出 DAG 的应用场景：

- 类继承关系是否有循环定义
- 操作系统中线程等待如何调度
- 课程先修关系
- 教学知识点安排
- 工程施工依赖
- 自动回复或转发链条是否成环

这些问题的共同本质是：

> 关系中有“先于 / 依赖于”含义

因此我们希望找到一个线性顺序，使得：

- 所有依赖都指向后面
- 即没有任何边指向一个更早的前驱顶点

这就是拓扑排序。

---

## 22.2 拓扑排序的定义

课件第 78 页给出：

给定有向图 `G`，尝试将所有顶点排成一个线性序列，使其次序与原图相容：

- 每个顶点都不会通过边指向其前驱顶点

若做不到，说明原图有环，不是 DAG。

---

## 22.3 偏序与全序

课件第 79 页指出：

- DAG 对应一个偏序集
- 拓扑排序就是构造一个与该偏序兼容的全序

这句话很抽象，但很重要。  
它说明拓扑排序不是“唯一真理顺序”，而只是：

> **众多合法线性扩展中的任意一个。**

因此：

- 一个 DAG 往往不止一种拓扑排序
- 只要依赖关系不唯一决定次序，就会有多种答案

---

## 23. 零入度算法（Kahn Algorithm）

## 23.1 核心定理：DAG 必有零入度顶点

课件第 80 页指出：

> 任何有向无环图中，必有一个零入度顶点。

直观上理解：

- 若每个顶点入度都至少为 1
- 就能不断逆着前驱回溯
- 有限图中最终必形成环
- 与 DAG 矛盾

这一定理直接导出拓扑排序算法。

---

## 23.2 算法思想

课件第 81 页给出零入度算法：

1. 找出所有入度为 0 的顶点，放入栈或队列 `S`
2. 反复：
   - 取出一个零入度顶点 `v`
   - 输出它
   - 删除 `v` 及其所有出边
   - 若某邻接点入度降为 0，则入栈 / 入队
3. 若最后所有顶点都被输出，则成功
4. 若仍有残留顶点，则说明图中存在环

这就是最经典的 **Kahn 算法**。

---

## 23.3 复杂度

课件第 81 页直接写出：

总体复杂度为：

$
O(n + e)
$

因为：

- 每个顶点至多进出栈 / 队列一次
- 每条边也只在其尾顶点被删除时处理一次

因此拓扑排序同样是线性时间可做的。

---

## 24. 基于 DFS 的拓扑排序

## 24.1 逆序输出零出度顶点

课件第 84 页给出另一种思路：

- 对图做 DFS
- 每当顶点被标记为 `VISITED` 时，就压入栈 `S`
- 一旦发现后向边，则立即报告 `NOT_A_DAG`
- DFS 全部结束后，再依次弹出 `S`

最终得到的就是拓扑排序。

---

## 24.2 为什么按 `fTime` 逆序就是拓扑序

因为在 DAG 中不存在后向边。  
对任意边 `(u,v)`：

- `u` 完成访问的时刻一定晚于 `v`
- 因而：

$
fTime(u) > fTime(v)
$

所以按 `fTime` 逆序排列，必然使：

- `u` 出现在 `v` 前面

这正与拓扑序要求一致。

---

## 24.3 代码框架

课件第 86～87 页给出 `TSort()`：

```cpp
bool TSort(Rank v, int& clock, Stack<Tv>* S) {
    dTime(v) = ++clock;
    status(v) = DISCOVERED;
    for (Rank u = firstNbr(v); u < UINT_MAX; u = nextNbr(v, u))
        switch (status(u)) {
            case UNDISCOVERED:
                parent(u) = v;
                type(v, u) = TREE;
                if (!TSort(u, clock, S)) return false;
                break;
            case DISCOVERED:
                type(v, u) = BACKWARD;
                return false;
            default:
                type(v, u) = dTime(v) < dTime(u) ? FORWARD : CROSS;
                break;
        }
    status(v) = VISITED;
    S->push(vertex(v));
    return true;
}
```

这段代码的逻辑非常漂亮：

- 一边 DFS
- 一边判环
- 一边累积逆后序序列

若发现 BACKWARD 边，则立刻说明图不是 DAG。

## 25. BFS 与 DFS 的对照理解

这一章最值得总结的，不是某段代码，而是 BFS 与 DFS 的差别。

### 25.1 BFS

- 使用队列
- 按层推进
- 天然适合最短距离
- 边主要分 TREE / CROSS
- 更像“波纹扩散”

### 25.2 DFS

- 使用递归 / 栈
- 一路深入后回溯
- 天然适合结构分析
- 有 TREE / BACKWARD / FORWARD / CROSS
- 更像“探洞 + 回退”

可以把它们理解为处理图的两种最基本视角：

> **BFS 关心“离起点多远”**  
> **DFS 关心“结构上谁包住谁、谁依赖谁、是否成环”**

## 26. 本章主线总结

### 26.1 图是比树更一般的关系模型

它允许：

- 多条路径
- 回路
- 多对多连接
- 有向 / 无向 / 带权等复杂结构

### 26.2 邻接矩阵与邻接表各有适用场景

- 邻接矩阵：
  - 边存在性判断快
  - 适合稠密图
- 邻接表：
  - 空间更省
  - 适合稀疏图与遍历

### 26.3 BFS 与 DFS 是一切图算法的基础

- BFS：层次扩张、最短路、分层结构
- DFS：递归深入、判环、边分类、拓扑排序

### 26.4 图遍历需要“状态”与“时间戳”

不像树遍历那样天然无歧义，图遍历必须显式维护：

- `UNDISCOVERED / DISCOVERED / VISITED`
- `dTime / fTime`
- `parent`
- `type`

### 26.5 DAG 与拓扑排序把“图遍历”变成“依赖管理”

无论是零入度算法还是 DFS 逆后序，本质都在回答：

> **如何把偏序关系扩展成一个合法的线性执行顺序。**

> 图这一章真正重要的，不只是学会邻接矩阵、邻接表和两段遍历代码，而是理解：一旦结构中允许环、允许多路径、允许复杂依赖，我们就必须借助 BFS 与 DFS 这样的系统遍历框架，才能在图这种最一般的关系网络中提取出距离、连通性、回路、层次和顺序。
