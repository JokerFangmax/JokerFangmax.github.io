# 预训练语言模型笔记：从 ELMo、BERT、GPT 到 Scaling Law

## 1. 引言

在理解了 Transformer 的基本结构之后，一个更关键的问题是：

> Transformer 为什么能成为大语言模型时代的统一基础架构？

这个问题的答案，不只在于 Transformer 本身的表达能力，还在于它非常适合**大规模预训练**。  
现代大语言模型的能力，并不是单靠某个特定任务训练出来的，而是在海量文本上进行预训练，再配合后续对齐和强化学习逐步形成的。

这一讲的重点可以概括为四条主线：

1. 大语言模型的核心训练目标是什么  
2. 语言模型通常经历哪些训练阶段  
3. 预训练模型的发展脉络是怎样的  
4. 为什么“规模化”会成为大模型时代最重要的方法论之一

---

## 2. 大语言模型的核心任务：Next Token Prediction

### 2.1 什么是 Next Token Prediction

大语言模型最核心的训练目标，通常可以概括成一句话：

> 给定前面的上下文，预测下一个 token。

也就是说，如果当前上下文是：

- `Tsinghua University is a`

模型的目标就是预测下一个最合理的 token，例如：

- `public`
- `university`

这种训练方式被称为 **next token prediction**，而逐个 token 生成文本的过程，则叫做 **autoregressive generation**。

### 2.2 为什么 Next Token Prediction 很重要

几乎所有自然语言任务，都可以统一为 next token prediction。

这是因为很多任务，本质上都可以写成“给定上下文，继续生成后续文本”的形式。例如：

#### 2.2.1 世界知识类任务

> The capital city of China is ___.

模型需要补出 `Beijing`。

#### 2.2.2 推理类任务

> If A then B, if B then not C. Now A = 1, so C = __.

模型需要完成逻辑推断。

#### 2.2.3 编程类任务

```python
def quick_sort(x: list) -> list:
    ____
```

模型需要补出合适的代码实现。

因此，从表面上看，语言模型是在做“预测下一个词”；但从更深层次看，它实际上是在做：

- 知识调用
- 模式建模
- 逻辑推断
- 结构补全
- 程序续写

### 2.3 语言建模为什么可以看作 World Modeling

Language Modeling = World Modeling。

这句话的含义是：

语言并不只是符号序列，它本身携带了大量关于世界的信息。  
如果一个模型在足够大的语料上进行语言建模，那么它在某种意义上就会被迫学习：

- 词语之间的语义关系
- 常识与事实知识
- 事件之间的统计规律
- 人类表达中的推理模式
- 各种任务的语言形式

因此，大语言模型的预训练可以看作一种极端形式的 multi-task learning：  
虽然训练目标始终是同一个——预测下一个 token，但它隐含覆盖了大量不同任务。

### 2.4 为什么语言建模天然适合规模化

语言建模几乎不需要额外标注，只需要大规模原始文本。

这意味着它有几个天然优点：

#### 2.4.1 不需要人工标注

训练标签就来自文本本身，因为“下一个 token”本来就包含在原始语料里。

#### 2.4.2 不要求统一格式

书籍、网页、代码、HTML、LaTeX、论坛文本都可以拿来训练。

#### 2.4.3 不需要复杂结构解析

不必先做句法树、语义图等显式结构分析，只需把文本转成 token 序列即可。

## 3. 大语言模型的训练阶段

### 3.1 三阶段训练框架

大语言模型的训练通常可以概括为三个主要阶段：

1. **Self-Supervised Pre-training**
2. **Supervised Fine-Tuning**
3. **Reinforcement Learning from Human Feedback**

这三步分别对应：

- 学知识
- 学会当助手
- 学会更好地对齐人类偏好

### 3.2 第一阶段：Self-Supervised Pre-training

#### 3.2.1 核心目标

第一阶段使用的是**大规模无标注文本**。  
训练方式就是 next token prediction：给模型一个上下文，让它预测接下来的 token，然后根据预测误差更新参数。

#### 3.2.2 这一阶段学到了什么

这个阶段最主要学到的是：

- 语义知识
- 领域知识
- 世界知识
- 语言统计规律
- 各类文本模式

也就是说，**预训练阶段主要负责“把知识灌进模型里”**。

#### 3.2.3 这一阶段的局限

这一阶段训练出来的模型虽然可以流畅续写文本，但它还不知道如何真正把这些知识用于回答用户问题。

换句话说，这个阶段的模型更像一个“很会续写的大文本预测器”，而不是一个成熟的 AI assistant。

### 3.3 第二阶段：Supervised Fine-Tuning

#### 3.3.1 为什么需要 SFT

即使模型已经在预训练中学到了很多知识，它也未必知道：

- 用户到底想问什么
- 应该用什么风格回答
- 什么时候应该拒答
- 什么样的回答才更像一个对话助手

因此需要一个额外阶段，让模型去学习**人类希望它如何回答问题**。

#### 3.3.2 SFT 的训练方式

SFT 使用的是**人工标注的问答或对话数据**。  
训练时，模型看到用户输入，然后学习模仿标注者写出的高质量回答。

#### 3.3.3 SFT 的作用

SFT 主要让模型学会：

- 理解 instruction
- 识别用户意图
- 组织回答格式
- 避免明显不当回答
- 在知识应用上更贴近真实使用场景

#### 3.3.4 SFT 的局限

一个问题往往有多种合理答案，但监督学习通常只给出其中一种。

这会带来两个风险：

1. 模型可能被迫模仿单一答案风格  
2. token-level 的监督可能让模型过于拘泥于表面形式，而不是答案质量本身

### 3.4 第三阶段：RLHF

#### 3.4.1 为什么要引入 RLHF

在很多问题上，我们其实很难说“标准答案唯一是什么”，但我们往往可以判断：

- 哪个回答更自然
- 哪个回答更有帮助
- 哪个回答更安全
- 哪个回答更符合人类偏好

于是，第三阶段不再直接给模型标准答案，而是给出**质量反馈**。

#### 3.4.2 RLHF 的基本思想

不再给模型逐字逐句的参考答案，而只给输出质量的反馈分数，并训练模型提高这些分数。

它的本质是：

- 模型先生成回答
- 人类或奖励模型判断回答好坏
- 模型学习提高“好回答”的概率

#### 3.4.3 RLHF 的作用

RLHF 的目标不是让模型学更多知识，而是：

- 更好对齐人类偏好
- 让回答更有帮助、更自然
- 让模型减少不当输出
- 进一步释放模型已有能力

### 3.5 额外趋势：Large-Scale Reinforcement Learning

一个更新的趋势是使用大规模强化学习训练 LLM 的 reasoning 能力。

例如在数学推理或规则明确的问题中，可以通过：

- 让模型输出推理过程
- 根据最终答案是否正确给予奖励
- 奖励正确推理，惩罚错误推理

来进一步提升模型的“思考能力”。

## 4. 大语言模型的发展脉络

### 4.1 GPT-1：预训练 + 微调范式的开端

GPT-1 的重要性在于，它第一次比较明确地展示了这样一个范式：

1. 先在大规模无标注语料上做生成式预训练
2. 再在具体下游任务上做监督微调

它的关键意义不在于今天看来多强，而在于它奠定了：

> “先预训练，再适配下游任务”的主路线。

### 4.2 GPT-2：规模化开始显现力量

GPT-2 的关键变化是“做大”。

它让研究界开始明显看到一个趋势：

> 当语言模型足够大、数据足够多时，它会在没有显式监督的情况下学会很多任务能力。

也就是说，模型开始表现出某种“零样本”或“弱提示即完成任务”的能力。

### 4.3 GPT-3：In-Context Learning 的震撼

GPT-3 的意义更大，因为它展示了一个非常关键的新现象：

> 模型不一定需要参数更新，只通过上下文里的例子，就能学会当前任务。

这就是 **in-context learning**。

这说明语言模型从“需要显式任务适配”走向“只靠上下文就能理解任务形式”。

### 4.4 ChatGPT：从 Few-shot 到 Zero-shot Assistant

很多时候，用户甚至不需要给 in-context examples，模型就能理解你的意思并作答。

这背后其实就是前面讲的训练三阶段共同作用的结果：

- 预训练提供知识
- SFT 提供助手行为
- RLHF 提供对齐和偏好优化

### 4.5 GPT-4 与多模态扩展

语言模型的发展不再局限于纯文本，而是开始走向：

- 图像 + 文本
- 多模态统一建模
- 更强的推理与任务泛化

## 5. Towards AGI：统一化趋势

### 5.1 统一架构

过去不同领域往往用不同架构，而现在越来越多任务都开始向 Transformer 统一。

### 5.2 统一任务

过去的 AI 体系更像“一个任务一个模型”，而现在越来越接近：

> 一个统一的大模型，处理多种不同任务。

### 5.3 统一模态

只要一种信息能被序列化，它就有可能被大模型学习：

- 语言
- 图像
- DNA
- 行为序列
- 代码
- 数学表达式

## 6. 预训练语言模型的发展：ELMo、GPT、BERT、T5

### 6.1 ELMo：预训练的起点

#### 6.1.1 ELMo 的背景

在 Word2Vec 时代，词向量是 **context-free** 的。  
同一个词不管出现在什么上下文里，它的表示都是固定的。

#### 6.1.2 ELMo 的核心贡献

ELMo 的关键思想是：

> 让词表示依赖上下文，得到 contextualized word embeddings。

#### 6.1.3 ELMo 的特点

ELMo 的代表性特点包括：

1. **Contextualized**
2. **Deep**
3. **Character-based**

#### 6.1.4 ELMo 的意义

ELMo 的真正价值不只是提出了一个模型，而是证明了：

> 在大规模语料上做语言建模，可以显著提升 NLP 表示学习。

### 6.2 GPT：Decoder-Only 预训练

#### 6.2.1 GPT 的基本思路

GPT 采用的是：

- **Transformer**
- **left-to-right language modeling**
- **decoder-only architecture**

也就是说，GPT 直接沿用最自然的语言建模目标：

\[
P(x_1, x_2, ..., x_n) = \prod_t P(x_t \mid x_{<t})
\]

#### 6.2.2 GPT 的优势

这种做法的优点在于：

- 目标简单直接
- 与自然文本生成一致
- 训练目标与推理形式一致
- 容易扩展到超大规模

#### 6.2.3 Attention Mask 的作用

其作用是：

- 训练时，整段序列可以一次前向传播计算所有位置的 loss
- 但每个位置只能看到左边上下文，不能看到未来 token

因此：

- **Training**：1 次 forward pass 就能算整段序列  
- **Inference**：生成长度为 \(N\) 的文本时，需要逐步生成，因此常常是 \(N\) 次 forward pass

### 6.3 BERT：Encoder-Only 预训练

#### 6.3.1 为什么要做 BERT

传统语言模型通常只使用左侧上下文，但语言理解往往需要双向上下文。

#### 6.3.2 为什么不能直接做双向语言模型

这里有两个困难：

1. 语言模型要定义合法概率分布，因此天然适合单向生成  
2. 在双向 encoder 中，一个词如果能直接看到自己，就会出现“答案泄漏”

#### 6.3.3 BERT 的解决方案：Masked LM

BERT 的核心思路是：

> 随机 mask 掉一部分输入词，然后预测这些被 mask 的词。

这样做的结果是：

- 模型可以同时看左右文
- 但又不会直接看到被预测的词本身

#### 6.3.4 Next Sentence Prediction

BERT 训练中还包含 **Next Sentence Prediction (NSP)**，用于帮助模型学习句间关系。

#### 6.3.5 BERT 的意义

BERT 最大的影响在于，它让研究界更强烈地认识到：

> 预训练 + 更大模型 = 更强性能。

### 6.4 T5：Encoder-Decoder 与 Text-to-Text 统一

#### 6.4.1 T5 的核心思想

T5 的出发点非常优雅：

> 把所有 NLP 任务统一写成 text-to-text。

例如：

- 翻译：输入英文，输出中文
- 摘要：输入原文，输出摘要
- 分类：输入句子，输出标签文字
- 问答：输入问题和背景，输出答案

#### 6.4.2 T5 的预训练方式

T5 的预训练会：

- 用 encoder 编码输入
- 用 decoder 预测输出
- 在原文中做 **mask spans**
- 预测被 mask 掉的内容

#### 6.4.3 T5 的意义

T5 的重要贡献在于它说明了：

> 各类 NLP 任务本质上都可以统一成生成任务。

### 6.5 三种预训练范式的对比

#### 6.5.1 Encoder-Only

更偏向表示学习和理解任务，适合理解，但不天然适合自由生成。

#### 6.5.2 Decoder-Only

更偏向自回归生成，训练目标自然、生成一致性强、扩展性好，因此现在是主流。

#### 6.5.3 Encoder-Decoder

更适合显式输入-输出映射的 text-to-text 任务。

## 7. 为什么现在主流是 Autoregressive

### 7.1 原因一：目标最自然

自然语言本身就是一个从前到后逐步展开的序列，因此 next token prediction 是最自然的训练目标。

### 7.2 原因二：训练目标与生成任务一致

对于对话、续写、代码生成、长文本创作等任务，模型最终都要“一个 token 一个 token 地往后写”。  
decoder-only 恰好与这种形式严格一致。

### 7.3 原因三：更容易规模化

参数规模、数据规模、序列长度都在不断扩大，不可扩展的方法最终会被淘汰。  
而 autoregressive decoder-only 模型在工程上通常最适合持续扩展，因此逐渐成为主流。

## 8. Scaling Law

### 8.1 什么是 Scaling Law

Scaling law 研究的是这样一个问题：

> 当模型规模、数据规模、计算量持续增加时，模型性能会如何变化？

也就是说，性能提升并不是完全随机的，而往往遵循某种可预测规律。

### 8.2 为什么 Scaling Law 很重要

训练一个超大模型非常昂贵，因此不可能靠蛮力做大规模 grid search。

所以我们需要知道：

- 小模型实验结果是否能预测大模型趋势
- 参数量和数据量该如何平衡
- 固定算力下最优配置大概在哪里

### 8.3 Power Law 关系

在很多任务中，研究者发现：

> 测试 loss 与训练数据规模之间存在 power-law 关系。

简单理解就是：

- 数据变多，loss 会下降
- 模型变大，loss 也通常会下降
- 算力增加，性能也会变好

### 8.4 Predictable Scaling

可以用成本较低的小规模实验，去预测大规模模型训练后的表现。

这在工业界极其重要，因为它意味着：

- 不需要每次都真训练一个超大模型
- 可以先做一批小模型实验
- 拟合 scaling curve
- 再推断目标大模型是否值得训练

### 8.5 Scaling Law 的工程意义

#### 8.5.1 指导模型规模设计

帮助判断在既定预算下，应该做更大的模型还是喂更多数据。

#### 8.5.2 指导数据与参数的配比

不是参数越大越好，也不是数据越多越好，而是要看两者是否匹配。

#### 8.5.3 指导训练资源规划

让团队在真正投入巨额算力之前，对预期性能有更合理估计。

## 9. 总结

### 9.1 这一讲的核心主线

#### 9.1.1 大语言模型的核心任务是 Next Token Prediction

虽然形式上只是预测下一个 token，但它实际上统一了承载知识、推理和任务表达的语言建模过程。

#### 9.1.2 LLM 的能力形成通常分三步

- 预训练：学知识  
- SFT：学会按照用户意图回答  
- RLHF：学会更符合人类偏好地回答  

#### 9.1.3 预训练模型经历了明显的发展脉络

- ELMo：上下文化表示  
- GPT：decoder-only 自回归预训练  
- BERT：encoder-only 双向理解  
- T5：统一成 text-to-text  

#### 9.1.4 最终主流走向了 autoregressive

因为它训练目标自然、生成形式一致、最适合大规模扩展。

#### 9.1.5 大模型时代的关键方法论是 scaling

性能提升不是偶然的，而是与模型规模、数据规模和算力投入之间存在可预测关系。

### 9.2 一句话总结

> 大语言模型的崛起，本质上是“自监督语言建模 + Transformer 架构 + 大规模扩展”三者共同作用的结果；而 ELMo、BERT、GPT、T5 则是这一演化过程中的几个关键里程碑。
