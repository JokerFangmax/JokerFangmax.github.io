# 为什么生成模型会用微分方程？Flow 与 Diffusion 的统一视角

上一篇我们已经讲清楚了一件事：

> 生成模型的本质，是学习一个数据分布，然后从这个分布中采样。

但这马上会带来下一个问题：

> 模型到底要怎样“采样”？
> 它怎么可能从一个复杂的数据分布里直接抽样出结果？

现代生成模型中，一个非常核心的想法是：

**不要直接从复杂分布里采样，而是先从一个很简单的分布出发，再通过一个连续演化过程，把它一步步变成目标数据分布。** 

这就是 flow model 和 diffusion model 的共同出发点。

它们乍一看很不一样：

- flow model 常写成 ODE
- diffusion model 常写成 SDE
- 一个看起来是确定性的
- 一个看起来是随机的

但如果从更高层次看，它们其实在做的是同一件事：

> **定义一个随时间演化的动态系统，让简单分布在时间推进中逐渐变成复杂分布。**

这一篇就来把这个统一视角讲清楚。

---

# 1. 从“静态采样”到“动态生成”

假设目标是生成一张真实图片。  
如果直接让模型在高维图像空间里“一步采样到位”，这会非常困难，因为目标分布 $p_{\text{data}}$ 往往极其复杂。

于是一个更自然的想法是：

1. 先从一个简单分布中采样一个初始点，比如标准高斯噪声
2. 然后让这个点随着时间逐步演化
3. 到最终时刻，它就落在真实数据分布所在的区域

也就是说，我们把原来“直接采样”的问题改写成：

> **设计一个从简单分布到复杂分布的连续变换过程。**

讲义中对 flow model 和 diffusion model 的定义，正是围绕这个思路展开的：先给一个容易采样的初始分布 $p_{\text{init}}$，通常取标准高斯，再通过 ODE 或 SDE 的演化，让终点分布逼近 $p_{\text{data}}$。

---

# 2. ODE 视角：Flow Model 是怎样工作的？

## 2.1 向量场：每个位置该往哪里走

讲义先从常微分方程（ODE）开始。  
在这里最核心的对象不是“样本”本身，而是一个 **向量场（vector field）**：

$
u : \mathbb{R}^d \times [0,1] \to \mathbb{R}^d,\quad (x,t)\mapsto u_t(x)
$

它的含义非常直观：

- 给定当前时刻 $t$
- 给定当前位置 $x$
- 向量场 $u_t(x)$ 会告诉你当前速度：“此时此刻应该朝哪个方向移动，以及移动得多快” 

可以把它想象成一张地图：

- 空间里每个位置都画了一根箭头
- 箭头的方向表示下一步该往哪走
- 箭头的长度表示移动速度大小

于是，一个点如果顺着这些箭头不断移动，就会形成一条轨迹。

---

## 2.2 ODE：轨迹如何跟随向量场演化

在这个框架下，一条轨迹 {$X_t$} 被定义为满足下面方程的函数：

$
\frac{d}{dt}X_t = u_t(X_t), \qquad X_0 = x_0
$

它的意思是：

- $X_t$ 是时刻 $t$ 的位置
- 它对时间的导数，就是当前位置上的向量场值
- 也就是说，轨迹始终沿着向量场给出的方向前进 

如果用最直观的话来说：

> **ODE 就是在说：当前位置怎么动，由当前所在位置和时间决定。**

这时候，“生成一个样本”就不再是凭空从复杂分布里跳出来，而变成：

> 从某个简单分布中的初始点 $x_0$ 出发，沿着 ODE 指定的轨迹一直走到复杂分布中的终点(采样点)。

---

## 2.3 Flow：从初始点到终点的整体变换

讲义进一步引入了 **flow** 的概念。  
如果把某个初始点 $x_0$ 在时间 $t$ 推进后得到的位置记为 $\psi_t(x_0)$，那么这个 $\psi_t$ 就叫 flow。它满足：

$
\frac{d}{dt}\psi_t(x_0)=u_t(\psi_t(x_0)), \qquad \psi_0(x_0)=x_0
$

因此，对于给定初值 $X_0=x_0$，轨迹可以写成：

$
X_t = \psi_t(X_0)
$

讲义里特别强调了一点：

> 向量场、ODE、flow，本质上是同一个动态系统的三种描述方式。  
> 向量场定义了 ODE，而 ODE 的解就是 flow。

这个观点非常重要，因为很多初学者会把这些概念分得太开。  
其实可以这样记：

- **向量场**：局部规则，告诉你每点怎么动
- **ODE**：运动方程，描述轨迹如何随时间变化
- **flow**：全局映射，告诉你起点最后被送到哪里

---

# 3. Flow Model：如何把 ODE 变成生成模型？

到这里还只是数学动力系统。  
那它怎么变成生成模型呢？

答案很简单：

> **让初始点本身是随机的。**

讲义指出，ODE 本身是确定性的：同样的初值一定会得到同样的轨迹。  
所以如果想让生成结果有随机性，就必须让初始条件 $X_0$ 来自一个随机分布 $p_{\text{init}}$，最常见的选择就是标准高斯 $\mathcal N(0,I)$。

于是 flow model 就可以写成：

$
X_0 \sim p_{\text{init}}, \qquad \frac{d}{dt}X_t = u_t^\theta(X_t)
$

其中 $u_t^\theta$ 是神经网络参数化的向量场。我们的目标是让最终时刻的样本满足：

$
X_1 \sim p_{\text{data}}
$

也就是说：

- 一开始从简单分布采样噪声
- 再通过神经网络定义的 ODE 持续演化
- 最终把这个噪声变成真实数据分布中的样本 

这就是 flow model 的本质。

---

## 3.1 一个容易忽略但很重要的点：网络学的是向量场，不是 flow 本身

讲义中特别提醒：

> 虽然叫 flow model，但神经网络参数化的其实是 **vector field**，不是 flow 本身。  
> flow 需要通过数值积分算出来。

这句话非常关键。

很多人一开始会误以为模型直接输出“从噪声到图像”的映射。  
其实不是。模型学到的是：

$
u_t^\theta(x)
$

也就是“在时刻 $t$，位于 $x$ 时该朝哪里走”。

而真正从 $X_0$ 到 $X_1$ 的完整变换，需要把这个局部移动规则一步步积分起来。

所以 flow model 的采样本质上是一个**数值求解 ODE** 的过程。

---

## 3.2 Euler 方法：最简单的 ODE 采样方式

***如果 ODE 很难直接解析求解，我们就用数值方法。*** ???  
讲义中最先给出的就是 Euler 方法，它的更新形式是：

$
FIXME: X_{t+h} = X_t + h\,u_t(X_t) 
$

其中 $h$ 是步长。它的意义很直观：

- 在当前位置 $X_t$
- 看一下当前向量场 $u_t(X_t)$
- 然后沿这个方向走一个很小的距离 $h$ 

所以 flow model 的采样流程就是：

1. 从 $p_{\text{init}}$ 采一个初始噪声 $X_0$
2. 把时间从 $0$ 划分成很多小步
3. 每一步按 Euler 规则更新
4. 最后返回 $X_1$

这就是“从噪声逐步变成样本”的最基础版本。

---

# 4. SDE 视角：为什么还要引入随机性？

如果 ODE 已经能做生成，为什么还要再讲 SDE？

因为 ODE 里的演化是**确定性的**。  
一旦初值给定，后面的轨迹就完全固定了。

但很多时候，我们希望生成过程本身也带有随机性。  
这时就要把 ODE 扩展成 **随机微分方程（SDE）**。

讲义中写道，SDE 是在 ODE 的基础上加入了随机轨迹，也就是 stochastic process。对于同一个初始条件，多次模拟可能会得到不同的结果。

这意味着：

> ODE 只在初始时刻注入随机性，之后演化是确定的；  
> SDE 则不仅初始是随机的，演化过程中也会持续加入随机扰动。

---

## 4.1 Brownian Motion：连续时间下的随机游走

SDE 的随机性通常通过 **布朗运动（Brownian motion）** 来建模。  
讲义把它描述成一种连续时间的随机游走：增量服从高斯分布，而且不同时段的增量彼此独立。

虽然严格的随机分析定义比较复杂，但在生成模型里，你可以先抓住这个直觉：

> Brownian motion 就像“每个极小时间间隔里都加入一点独立高斯噪声”。

这样就很容易理解后面的 SDE 形式了。

---

## 4.2 SDE：在“朝某方向移动”的同时，不断加随机噪声

讲义给出的 SDE 形式是：

$
dX_t = u_t(X_t)\,dt + \sigma_t\,dW_t
$

这里有两部分：

1. **漂移项（drift）**：$u_t(X_t)\,dt$  
   表示仍然按照向量场方向移动

2. **扩散项（diffusion）**：$\sigma_t\,dW_t$  
   表示叠加一个随机扰动，强度由 $\sigma_t$ 决定 

所以 SDE 可以理解成：

> **一边被向量场“推着走”，一边不断受到随机噪声扰动。**

如果把 ODE 比作一辆严格按照导航行驶的车，那么 SDE 更像是：

- 车仍然有导航方向
- 但一路上会不断受到风、路面抖动、随机干扰的影响

这使得轨迹不再完全确定。

---

## 4.3 一个关键区别：SDE 一般没有确定性的 flow map

对于 ODE，只要初值确定，终点就确定，因此我们可以定义一个 flow map $\psi_t$。

但讲义指出，SDE 中通常已经没有这样的确定性 flow map，因为即使初始点相同，随机扰动不同，最后的结果也会不同。

这也是 flow model 和 diffusion model 最根本的结构差异之一：

- **Flow model**：给定初值后，轨迹确定
- **Diffusion model**：给定初值后，轨迹通常仍然是随机的

---

# 5. Diffusion Model：如何把 SDE 变成生成模型？

和 flow model 完全类似，diffusion model 也是从一个简单初始分布开始：

$
X_0 \sim p_{\text{init}}
$

然后让样本按一个 SDE 演化：

$
dX_t = u_t^\theta(X_t)\,dt + \sigma_t\,dW_t
$

其中：

- $u_t^\theta$ 是神经网络参数化的向量场
- $\sigma_t$ 是预先设定好的扩散系数
- 目标仍然是让 $X_1 \sim p_{\text{data}}$ 

讲义中甚至明确总结道：

> diffusion model 由一个神经网络向量场 $u_t^\theta$ 和一个固定扩散系数 $\sigma_t$ 组成；  
> 生成时，从简单分布初始化，然后模拟该 SDE 到终点。

所以从最高层次看，diffusion model 和 flow model 的结构几乎完全一样：

- 都有简单初始分布
- 都有神经网络定义的动态规则
- 都通过时间演化把初始分布送到数据分布

它们最大的区别，只在于**是否在演化过程中持续注入噪声**。

---

# 6. Euler-Maruyama：SDE 的最基础采样方法

就像 ODE 需要 Euler 方法一样，SDE 也需要数值模拟方法。  
讲义中给出的最基础方法是 **Euler-Maruyama**：

$
X_{t+h} = X_t + h\,u_t(X_t) + \sqrt{h}\,\sigma_t \epsilon_t,\qquad \epsilon_t\sim\mathcal N(0,I)
$

这个式子和 Euler 方法几乎一模一样，只是多了一项随机噪声。

它很好理解：

- $X_t + h\,u_t(X_t)$ 是按向量场做的正常推进
- $\sqrt{h}\,\sigma_t \epsilon_t$ 是这一小步中新加入的高斯随机扰动

所以 diffusion model 的采样流程就是：

1. 从 $p_{\text{init}}$ 采样 $X_0$
2. 对每个时间步：
   - 计算当前漂移方向
   - 采一个高斯噪声
   - 同时做“确定性更新 + 随机更新”
3. 最后得到 $X_1$ 

从这个角度看，“扩散模型采样”其实就是在做一连串小步的随机数值模拟。

---

# 7. Flow 和 Diffusion 的统一视角

现在我们就可以把两者放在一起看了。

## 7.1 共同点

无论是 flow 还是 diffusion，它们都在做下面这件事：

- 从简单分布 $p_{\text{init}}$ 出发
- 定义一个随时间变化的动态系统
- 让样本在这个系统下逐步演化
- 希望终点分布逼近 $p_{\text{data}}$ 

所以从宏观角度说，它们都是：

> **把生成问题转化成分布随时间演化的问题。**

---

## 7.2 不同点

差别主要有两点。

### 第一，轨迹是否确定

- 在 flow model 中，给定初值 $X_0$ 后，轨迹由 ODE 唯一决定 
- 在 diffusion model 中，即使给定初值，后续轨迹也因为随机噪声而不唯一 

### 第二，是否持续注入噪声

- flow model：随机性只来自初始采样
- diffusion model：随机性不仅来自初值，还来自整个演化过程中的扩散项 

---

## 7.3 一个很漂亮的统一结论

讲义里还有一句非常值得记住的话：

> **A diffusion model with $\sigma_t = 0$ is a flow model.** 

这句话的意思是：

如果把 diffusion model 里的噪声项关掉，也就是令扩散系数 $\sigma_t = 0$，那么 SDE 就退化成 ODE，于是 diffusion model 就变成了 flow model。

所以 flow model 其实可以看成 diffusion model 的一个特殊情形。

这让两者之间的关系一下变得非常清晰：

> **Flow 是“无噪声的扩散”；Diffusion 是“带噪声的流动”。**

---

# 8. 为什么这种建模方式对生成特别自然？

因为在生成任务里，我们最熟悉的简单分布就是高斯噪声，而最难的部分是数据分布本身很复杂。

如果强行一步从噪声跳到数据，会很难学。  
但如果允许模型做很多个很小的连续变化，每一步只做一点点调整，那么问题就自然多了。

这其实和很多数值方法、最优化方法的思想非常一致：

- 大变化难以直接完成
- 但拆成许多小变化后，就更容易建模和求解

flow / diffusion 正是把这个思想用到了概率分布上：

> 不要求模型一次学会“从噪声直接生成图像”，  
> 而是只要求它学会“当前这一步该怎样稍微改一下”。

后面无论是 flow matching，还是 score matching，本质上都是在学习这种“局部更新规则”。

# 9. 本章小结

这一篇最核心的内容可以总结成下面几条：

1. 生成模型可以不直接从复杂分布采样，而是从简单分布出发，通过连续动态过程变换到目标分布 
2. 在 ODE 视角下，动态系统由向量场 $u_t(x)$ 定义；轨迹满足  
   $
   \frac{d}{dt}X_t = u_t(X_t)
   $
   flow 是这个 ODE 的解映射 
3. Flow model 的核心是：初始随机、演化确定；网络学的是向量场，不是 flow 本身 
4. 在 SDE 视角下，动态过程多了一个随机扩散项  
   $
   dX_t = u_t(X_t)dt + \sigma_t dW_t
   $
   因此演化本身也是随机的 
5. Diffusion model 和 flow model 的统一点在于：它们都在做“简单分布 $\to$ 数据分布”的时间演化；当 $\sigma_t=0$ 时，diffusion 就退化成 flow
