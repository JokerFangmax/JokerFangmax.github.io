## 1. 从 ADT 到数据结构：为什么先讲“向量”

### 1.1 抽象数据类型（ADT）与数据结构的区别

课件一开始就区分了两个容易混淆的概念：

- **抽象数据类型（Abstract Data Type, ADT）**
- **数据结构（Data Structure）**

它们的差别可以概括为：

#### 1.1.1 ADT 关注“做什么”

ADT 是对某类数据对象及其操作的**抽象定义**。  
它强调的是：

- 这种数据对象是什么；
- 支持哪些操作；
- 各种操作的语义是什么。

此时通常**不关心具体如何存储**，也不强调某种实现的时间复杂度。

#### 1.1.2 数据结构关注“怎么做”

数据结构是在某种具体语言和存储机制下，对 ADT 的一整套实现。  
它强调的是：

- 数据在内存中如何组织；
- 每个操作怎么写；
- 每个操作的时间、空间复杂度如何。

所以可以这样理解：

> **ADT 是接口和规范，数据结构是接口背后的具体实现。**

同一个 ADT，往往可以有多种不同的数据结构实现。  
例如“线性表”可以用：

- 顺序存储（数组 / 向量）
- 链式存储（链表）

来实现，而不同实现的代价与适用场景并不相同。

### 1.2 为什么 ADT 很重要

ADT 的意义在于，它把“使用者”和“实现者”解耦了。

- **高层应用**只需要调用统一接口；
- **底层实现**可以不断优化，而不影响上层逻辑。

从软件工程角度看，这种抽象带来几个直接好处：

1. **分工清晰**：算法设计者和数据结构实现者可以协作。
2. **可替换性强**：接口不变时，可以替换底层实现。
3. **复用性高**：同一数据结构实现一次，可供多个算法复用。
4. **代码更短、更稳**：减少重复造轮子。

## 2. 从数组到向量：为什么不直接用数组

### 2.1 数组的本质

在 C/C++ 中，数组本质上是一段**地址连续**的内存。  
数组元素与下标一一对应，例如：

- `A[0], A[1], A[2], ..., A[n-1]`

若每个元素占用 `s` 字节，则：

\[
A[i] 	ext{ 的地址 } = A + i \cdot s
\]

因此数组支持**按秩（rank）直接访问**，随机访问复杂度是：

\[
O(1)
\]

这也是数组最大的优势。

### 2.2 数组的局限

尽管数组高效，但直接使用原生数组仍存在问题：

1. **缺乏封装**：很多操作需要自己写，易出错。
2. **容量固定**：长度通常在分配时就确定。
3. **接口不统一**：插入、删除、查找、去重等都要手写。
4. **类型扩展性差**：复杂元素类型管理不够方便。
5. **安全性不足**：越界、野指针、内存管理都容易出问题。

### 2.3 向量的定义

向量可以视为数组的抽象与泛化：

> **向量 = 一组按线性次序组织的元素，并以“秩（rank）”作为统一访问方式的封装结构。**

也就是说：

- 每个元素都有一个秩 `r ∈ [0, n)`；
- 可以通过秩直接访问；
- 但在此基础上又支持插入、删除、扩容、遍历等更高级操作。

因此，向量兼顾了：

- 数组的顺序存储与高效随机访问；
- 更安全、更统一、更强大的接口封装。

## 3. 向量 ADT：接口设计

### 3.1 基本访问接口

#### 3.1.1 `size()`

返回当前向量中的元素个数，即规模 `n`。

#### 3.1.2 `get(r)`

获取秩为 `r` 的元素。

#### 3.1.3 `put(r, e)`

把秩为 `r` 的元素替换为 `e`。

### 3.2 结构修改接口

#### 3.2.1 `insert(r, e)`

把元素 `e` 插入到秩 `r` 处，原来 `[r, n)` 的元素整体后移一位。

#### 3.2.2 `remove(r)`

删除秩为 `r` 的元素，并返回其原值。

#### 3.2.3 `remove(lo, hi)`

删除区间 `[lo, hi)` 内的所有元素。

### 3.3 顺序与查找接口

#### 3.3.1 `disordered()`

判断当前向量是否已按非降序排列，或者统计其紊乱程度。

#### 3.3.2 `find(e)`

在**无序向量**中查找元素 `e`。

#### 3.3.3 `search(e)`

在**有序向量**中查找元素 `e`，其返回值通常比 `find` 更有语义价值。

### 3.4 处理接口

#### 3.4.1 `sort()`

对向量排序。

#### 3.4.2 `deduplicate()`

对无序向量去重。

#### 3.4.3 `uniquify()`

对有序向量去重。

#### 3.4.4 `traverse()`

对每个元素统一执行某个操作。

## 4. 向量的模板类实现框架

### 4.1 为什么要用模板类

向量不应该只服务于某一种元素类型。  
如果只支持 `int`，那就无法方便地用于：

- `double`
- `string`
- 自定义结构体
- 指针类型
- 复杂对象

因此最自然的设计是：

```cpp
template <typename T>
class Vector {
private:
    Rank _size;
    Rank _capacity;
    T* _elem;
protected:
    /* 内部函数 */
public:
    /* 构造、析构、接口 */
};
```

其中：

- `_size`：当前元素个数
- `_capacity`：当前容量
- `_elem`：底层连续存储区首地址

### 4.2 向量的核心不变量

一个合法的向量对象通常应满足这些条件：

1. `0 <= _size <= _capacity`
2. `_elem` 指向一段连续可用的存储空间
3. 有效元素存储在 `_elem[0 .. _size)` 中
4. 剩余 `_capacity - _size` 是保留空间

这几个条件后面在插入、删除、扩容、缩容时都必须被维护。

## 5. 构造、复制与析构

### 5.1 默认构造

默认构造函数通常为向量分配一个初始容量：

```cpp
Vector(int c = DEFAULT_CAPACITY) {
    _elem = new T[_capacity = c];
    _size = 0;
}
```

这说明：

- 向量一开始是空的；
- 但不是“没有空间”，而是已经预留了初始容量。

这样做比“每次插入一个元素就重新申请一次空间”要高效得多。

### 5.2 区间复制构造

构造函数还可以支持从数组区间或另一个向量区间复制：

```cpp
Vector(T const* A, Rank lo, Rank hi) { copyFrom(A, lo, hi); }
Vector(Vector<T> const& V, Rank lo, Rank hi) { copyFrom(V._elem, lo, hi); }
Vector(Vector<T> const& V) { copyFrom(V._elem, 0, V._size); }
```

这些重载说明一个重要思想：

> 向量不是“只能从空开始插入”，它也应该支持高效的批量初始化。

### 5.3 `copyFrom()` 的思想

典型实现如下：

```cpp
template <typename T>
void Vector<T>::copyFrom(T const* A, Rank lo, Rank hi) {
    _elem = new T[_capacity = max(DEFAULT_CAPACITY, 2 * (hi - lo))];
    for (_size = 0; lo < hi; _size++, lo++)
        _elem[_size] = A[lo];
}
```

这个函数做了两件事：

1. 按区间长度开辟足够空间；
2. 把源区间元素逐个复制到内部数组。

时间复杂度是：

\[
O(hi-lo)
\]

### 5.4 析构函数

```cpp
~Vector() { delete [] _elem; }
```

这一步非常关键，因为向量底层使用了动态内存。  
如果没有析构函数，对象销毁时就会造成内存泄漏。

## 6. 动态空间管理：为什么向量必须可扩容

### 6.1 静态空间管理的问题

如果底层数组容量固定，那么会遇到两个典型问题：

#### 6.1.1 上溢（overflow）

数组满了，但还要继续插入。  
即使系统仍有大量空闲内存，这个向量也无法再装新元素。

#### 6.1.2 下溢（underflow）

删除大量元素后，数组大部分空间都空着，但依然被占用。

### 6.2 装填因子（load factor）

向量的空间利用率通常用装填因子表示：

\[
\lambda = rac{_size}{_capacity}
\]

若 \(\lambda\) 很小，说明浪费空间较多；  
若 \(\lambda\) 太接近 1，则说明插入时很容易触发扩容。

因此需要在：

- **空间利用率**
- **扩容开销**

之间做平衡。

---

## 7. 扩容算法

### 7.1 基本实现

典型扩容代码如下：

```cpp
template <typename T>
void Vector<T>::expand() {
    if (_size < _capacity) return;
    _capacity = max(_capacity, DEFAULT_CAPACITY);
    T* oldElem = _elem;
    _elem = new T[_capacity <<= 1];
    for (Rank i = 0; i < _size; i++)
        _elem[i] = oldElem[i];
    delete [] oldElem;
}
```

### 7.2 扩容时发生了什么

当向量满了之后，扩容过程可以拆成四步：

1. 申请一块更大的新空间；
2. 把旧空间中的元素全部复制过去；
3. 释放旧空间；
4. 更新 `_elem` 和 `_capacity`。

### 7.3 为什么采用“容量加倍”

课件强调一个关键问题：

> 为什么扩容要“加倍”，而不是每次只加一个固定常数？

这是因为两种策略的分摊复杂度差异非常大。

---

## 8. 分摊分析：为什么倍增是对的

### 8.1 固定增量扩容的代价

若每次扩容都只增加固定容量 `I`，例如：

```cpp
_capacity += INCREMENT;
```

那么在连续插入很多元素时，扩容会发生得非常频繁。  
每次扩容都要复制已有元素，复制代价形成一个算术级数：

\[
0 + I + 2I + 3I + \cdots + (m-1)I = O(n^2)
\]

于是总扩容时间是平方级，平均到每次插入上，分摊成本是：

\[
O(n)
\]

### 8.2 倍增扩容的代价

若每次容量加倍：

```cpp
_capacity <<= 1;
```

则扩容只在容量达到：

\[
1, 2, 4, 8, 16, \dots
\]

这些节点上发生。  
每次复制代价形成几何级数：

\[
1 + 2 + 4 + 8 + \cdots + 2^{m-1} = O(n)
\]

因此总扩容代价是线性的。  
平均到每次插入操作，分摊复杂度就变成：

\[
O(1)
\]

### 8.3 为什么这里讲的是“分摊”而不是“平均”

#### 8.3.1 平均复杂度（average complexity）

平均复杂度通常假设：

- 各种情况按某种概率分布出现；
- 然后对其成本求期望。

#### 8.3.2 分摊复杂度（amortized complexity）

分摊复杂度不依赖概率模型，而是：

> 观察一长串连续操作的总成本，然后摊到每次操作头上。

所以，向量插入的经典结论是：

- **单次最坏情况**：扩容时可达 \(O(n)\)
- **分摊复杂度**：连续插入下是 \(O(1)\)

## 9. 元素访问：重载下标运算符

### 9.1 为什么要重载 `[]`

如果只能写：

- `get(r)`
- `put(r, e)`

那么使用起来不如数组直观。  
所以通常会重载 `[]`，让向量也支持：

```cpp
V[r]
```

### 9.2 可写与只读版本

典型实现有两个版本：

```cpp
T& operator[](Rank r) { return _elem[r]; }
const T& operator[](Rank r) const { return _elem[r]; }
```

第一个版本返回引用，因此可作为左值：

```cpp
V[r] = x;
```

第二个版本用于常量对象，只允许读取。

## 10. 无序向量的基本修改操作

### 10.1 插入 `insert(r, e)`

典型实现如下：

```cpp
template <typename T>
Rank Vector<T>::insert(Rank r, T const& e) {
    expand();
    for (Rank i = _size; i > r; i--)
        _elem[i] = _elem[i - 1];
    _elem[r] = e;
    _size++;
    return r;
}
```

### 10.2 插入的原理

插入到秩 `r` 的本质是：

1. 先为新元素腾出位置；
2. 把原 `[r, _size)` 的所有元素向右移动一格；
3. 再把新元素放到 `_elem[r]`。

### 10.3 插入复杂度

若插入位置是 `r`，则需要移动的元素个数约为：

\[
_size - r
\]

因此时间复杂度为：

\[
O(n-r)
\]

最坏情况发生在头部插入，复杂度是：

\[
O(n)
\]

### 10.4 区间删除 `remove(lo, hi)`

典型实现：

```cpp
template <typename T>
int Vector<T>::remove(Rank lo, Rank hi) {
    if (lo == hi) return 0;
    while (hi < _size) _elem[lo++] = _elem[hi++];
    _size = lo;
    shrink();
    return hi - lo;
}
```

### 10.5 区间删除的原理

删除区间 `[lo, hi)` 的方式不是“把中间挖空”，而是：

- 把后缀 `[hi, n)` 整体前移到 `lo` 开始的位置；
- 然后更新 `_size`。

### 10.6 区间删除复杂度

删除区间后，需要移动后缀长度约为：

\[
n - hi
\]

因此时间复杂度为：

\[
O(n - hi)
\]

### 10.7 单元素删除 `remove(r)`

单元素删除只是区间删除的特例：

```cpp
T Vector<T>::remove(Rank r) {
    T e = _elem[r];
    remove(r, r + 1);
    return e;
}
```

其复杂度仍是：

\[
O(n-r)
\]

### 10.8 为什么不能反过来用 `remove(r)` 实现区间删除

如果想用循环不断调用 `remove(r)` 来删除 `[lo, hi)`，那每次都会触发一次后缀前移，总复杂度可能退化为：

\[
O(n^2)
\]

所以一次性删除整个区间会更高效。

## 11. 无序向量的查找与去重

### 11.1 顺序查找 `find()`

无序向量中没有顺序信息可利用，因此只能顺序扫描。

典型实现：

```cpp
template <typename T>
Rank Vector<T>::find(T const& e, Rank lo, Rank hi) const {
    while ((lo < hi--) && (e != _elem[hi]));
    return hi;
}
```

### 11.2 顺序查找复杂度

顺序查找是**输入敏感**的：

- 最好情况：`O(1)`
- 最坏情况：`O(n)`

### 11.3 判等器与比较器

课件引入了 `Entry<K, V>` 模板类，并重载了：

- `==`
- `!=`
- `<`
- `>`

这体现了泛型设计中的一个重要思想：

> 算法只需要知道“如何比较”，而不必知道对象内部细节。

### 11.4 无序向量去重 `deduplicate()`

典型实现如下：

```cpp
template <typename T>
Rank Vector<T>::deduplicate() {
    Rank oldSize = _size;
    for (Rank i = 1; i < _size; )
        if (find(_elem[i], 0, i) < 0)
            i++;
        else
            remove(i);
    return oldSize - _size;
}
```

### 11.5 去重原理

- 从左到右扫描每个元素 `_elem[i]`；
- 去前缀 `[0, i)` 中查找是否已经出现过它；
- 若没有出现，则保留；
- 若已经出现，则删掉当前位置的重复项。

### 11.6 去重复杂度

综合下来，最坏复杂度是：

\[
O(n^2)
\]

## 12. 遍历：统一处理所有元素

### 12.1 遍历接口的意义

遍历的核心思想是：

> 把“如何访问每个元素”封装在向量内部，把“对元素做什么”作为参数传进来。

### 12.2 函数指针版本

```cpp
template <typename T>
void Vector<T>::traverse(void (*visit)(T&)) {
    for (Rank i = 0; i < _size; i++) visit(_elem[i]);
}
```

### 12.3 函数对象版本

```cpp
template <typename T> template <typename VST>
void Vector<T>::traverse(VST& visit) {
    for (Rank i = 0; i < _size; i++) visit(_elem[i]);
}
```

### 12.4 例子：所有元素加一

```cpp
template <typename T>
struct Increase {
    virtual void operator()(T& e) { e++; }
};
```

再配合：

```cpp
V.traverse(Increase<T>());
```

即可把向量中所有元素统一加一。

## 13. 有序向量：为什么预排序很重要

### 13.1 有序向量的优势

无序向量的查找、去重等操作往往只能线性做。  
而一旦向量有序，很多操作都能显著优化，例如：

- 查找可用二分；
- 去重可线性完成；
- 插入位置可精确确定；
- 顺序性可被不变量利用。

### 13.2 如何判断是否有序

若一个序列中存在相邻元素 `A[i] > A[i+1]`，则它至少不是完全有序的。  
因此可以通过扫描相邻对来检查顺序性。

---

## 14. 有序向量去重 `uniquify()`

### 14.1 低效版

朴素做法是：一旦相邻两个元素相等，就删除后者。  
由于 `remove(i)` 会触发整体前移，这种做法最坏仍可能达到：

\[
O(n^2)
\]

### 14.2 高效版：双指针思想

高效版本如下：

```cpp
template <typename T>
int Vector<T>::uniquify() {
    Rank i = 0, j = 0;
    while (++j < _size)
        if (_elem[i] != _elem[j]) _elem[++i] = _elem[j];
    _size = ++i;
    shrink();
    return j - i;
}
```

### 14.3 高效去重的原理

因为向量已经有序，所以相等元素一定是连续出现的。  
于是我们只需：

- `j` 负责扫描每个元素；
- `i` 负责维护当前“已去重前缀”的最后位置。

这是一种**原地压缩**。

### 14.4 复杂度分析

时间复杂度是：

\[
O(n)
\]

空间复杂度是：

\[
O(1)
\]

## 15. 有序向量查找：从二分到插值

### 15.1 统一接口 `search(e, lo, hi)`

课件设计了一个统一接口 `search()`，底层可以随机调用：

- 二分查找
- Fibonacci 查找

这说明：

> 接口应该独立于具体算法实现。

## 16. 二分查找 A：经典三分支写法

### 16.1 基本思想

每一步取中点 `mi`，与目标 `e` 比较：

- 若 `e < A[mi]`：去左半边
- 若 `A[mi] < e`：去右半边
- 若相等：直接命中返回

### 16.2 实现

```cpp
template <typename T>
static Rank binSearch(T* S, T const& e, Rank lo, Rank hi) {
    while (lo < hi) {
        Rank mi = (lo + hi) >> 1;
        if (e < S[mi]) hi = mi;
        else if (S[mi] < e) lo = mi + 1;
        else return mi;
    }
    return -1;
}
```

### 16.3 复杂度

每一步都把问题规模减半，因此：

\[
T(n) = O(\log n)
\]

### 16.4 查找长度（Search Length）

更细致地评估查找算法时，可以看**关键码比较次数**，包括成功与失败查找的平均比较次数等。

---

## 17. Fibonacci 查找

### 17.1 动机

经典二分查找中，左、右分支的比较代价未必完全对称。  
Fibonacci 查找利用 Fibonacci 数的划分特性，使划分比例更适合某些常数开销模型。

### 17.2 性能特点

Fibonacci 查找的渐近复杂度仍然是：

\[
O(\log n)
\]

但在某些意义下，其平均查找长度在常数因子上可能略优于经典二分。

---

## 18. 二分查找 B 与 C：返回值语义的改进

### 18.1 为什么经典版不够好

经典版二分查找失败时直接返回 `-1`。  
但在很多应用中，我们更希望它告诉我们：

- 目标元素若不存在，应该插在哪里；
- 不大于 `e` 的最后一个元素在哪里；
- 大于 `e` 的第一个元素在哪里。

### 18.2 版本 C：扩展语义最优雅

版本 C 的代码如下：

```cpp
template <typename T>
static Rank binSearch(T* S, T const& e, Rank lo, Rank hi) {
    while (lo < hi) {
        Rank mi = (lo + hi) >> 1;
        e < S[mi] ? hi = mi : lo = mi + 1;
    }
    return lo - 1;
}
```

其返回值语义是：

> 返回 **不大于 `e` 的最后一个元素的秩**。

### 18.3 版本 C 的循环不变性

它依赖这样一个循环不变式：

\[
A[0, lo) \le e < A[hi, n)
\]

当循环结束时，`lo == hi`，此时 `lo - 1` 就是我们要找的边界位置。

## 19. 插值查找

### 19.1 基本思想

若数据是近似均匀分布的，那么目标值在区间中的相对位置，往往可以通过数值比例估计：

\[
mi = lo + (hi-lo)\cdot rac{e-A[lo]}{A[hi]-A[lo]}
\]

这就是插值查找。

### 19.2 性能特点

在理想均匀分布下，插值查找平均复杂度可以达到：

\[
O(\log\log n)
\]

### 19.3 为什么它没有成为主流

因为插值查找依赖较强的数据分布假设：

- 若分布不均匀，效果可能很差；
- 还需要乘法、除法；
- 容易被局部扰动影响。

因此工程中常见做法是混合使用插值、二分和顺序查找。

## 20. 排序接口：统一入口的重要性

课件给出一个统一排序入口，它可以选择不同排序算法。  
这是一种很好的设计思想：

> 用同一个接口抽象“排序”这一操作，而把具体算法放到底层实现。

## 21. 起泡排序

### 21.1 核心思想

- 若序列无序，则总存在某对相邻逆序元素；
- 扫描相邻元素并交换逆序对；
- 一趟扫描后，最大的元素一定会“浮”到末尾。

### 21.2 基本版

```cpp
template <typename T>
void Vector<T>::bubbleSort(Rank lo, Rank hi) {
    while (lo < --hi)
        for (Rank i = lo; i < hi; i++)
            if (_elem[i] > _elem[i + 1])
                swap(_elem[i], _elem[i + 1]);
}
```

### 21.3 正确性

循环不变式是：

> 经过 `k` 趟扫描后，最大的 `k` 个元素必然已经就位。

### 21.4 提前终止版

若某一整趟扫描中没有任何交换，说明序列已经有序，可以提前停止。  
最好情况可达到：

\[
O(n)
\]

### 21.5 稳定性

起泡排序是稳定的。  
因为它只交换**相邻逆序对**，相等元素不会跨过彼此。

### 21.6 复杂度

- 最好情况：`O(n)`
- 最坏情况：`O(n^2)`

## 22. 归并排序

### 22.1 分而治之思想

1. 把序列一分为二；
2. 分别递归排序；
3. 把两个有序子序列归并成一个有序序列。

### 22.2 递归框架

```cpp
template <typename T>
void Vector<T>::mergeSort(Rank lo, Rank hi) {
    if (hi - lo < 2) return;
    int mi = (lo + hi) >> 1;
    mergeSort(lo, mi);
    mergeSort(mi, hi);
    merge(lo, mi, hi);
}
```

### 22.3 二路归并的原理

若：

- 左半段 `[lo, mi)` 已有序；
- 右半段 `[mi, hi)` 已有序；

则只需反复比较两段当前最小元素，把较小者放入结果数组。

### 22.4 为什么归并只需线性时间

归并过程中总迭代次数不超过两段长度之和，因此时间复杂度是：

\[
O(n)
\]

### 22.5 归并排序复杂度

递推式为：

\[
T(n) = 2T(n/2) + O(n)
\]

所以：

\[
T(n) = O(n \log n)
\]

### 22.6 归并排序的优缺点

#### 优点

1. **最坏情况性能优秀**：始终是 `O(n log n)`
2. **稳定**
3. **顺序访问友好**
4. **适合链表、外部排序、并行化**

#### 缺点

1. **不是原地排序**
2. 需要额外辅助空间
3. 即使输入接近有序，也仍然需要 `O(n log n)` 时间

## 23. 位图（Bitmap）

### 23.1 位图的基本思想

位图用于表示一个**有限整数集合**。  
如果整数范围是 `[0, m)`，那么可以用一个 bit 数组来表示：

- 第 `k` 位为 1：表示 `k` 在集合中；
- 第 `k` 位为 0：表示 `k` 不在集合中。

### 23.2 为什么位图省空间

表示 `[0, m)` 的集合只需：

\[
m/8 	ext{ 字节}
\]

这在“整数范围不太大，但数据量极大”的场景中非常有吸引力。

### 23.3 位图的结构

典型实现：

```cpp
class Bitmap {
private:
    int N;
    char* M;
public:
    Bitmap(int n = 8) {
        M = new char[N = (n + 7) / 8];
        memset(M, 0, N);
    }
    ~Bitmap() { delete [] M; M = NULL; }
};
```

### 23.4 `test / set / clear` 的位运算原理

第 `k` 位属于第：

\[
k >> 3 = \lfloor k/8 
floor
\]

个字节；偏移是：

\[
k \& 0x07 = k mod 8
\]

于是：

```cpp
bool test(int k)  { return M[k >> 3] &  (0x80 >> (k & 0x07)); }
void set(int k)   { M[k >> 3] |=  (0x80 >> (k & 0x07)); }
void clear(int k) { M[k >> 3] &= ~(0x80 >> (k & 0x07)); }
```

## 24. 位图的典型应用

### 24.1 大数据去重

若有 `n` 个整数，取值范围在 `[0, m)`，且满足：

\[
m \ll n
\]

那么位图去重通常很高效：

```cpp
Bitmap B(m);
for (int i = 0; i < n; i++)
    B.set(A[i]);
for (int k = 0; k < m; k++)
    if (B.test(k)) {
        /* k 是去重后的元素 */
    }
```

复杂度：

- 时间：`O(n + m)`
- 空间：`O(m)`

### 24.2 埃拉托斯特尼筛法

位图的另一个经典应用是求素数表：

1. 先假设 `[2, n)` 中所有数都可能是素数；
2. 从 2 开始，把其倍数全部标记为合数；
3. 对每个未被标记的数重复此过程。

### 24.3 筛法优化

两个常见优化：

1. 内层从 `2*i` 改为 `i*i`
2. 外层从 `i < n` 改为 `i*i < n`

本质都是为了减少重复工作。

## 25. 位图的快速初始化：校验环思想

### 25.1 为什么初始化也是成本

常规位图构造时会调用：

```cpp
memset(M, 0, N);
```

其本质仍是逐字节清零，复杂度是：

\[
O(N)
\]

### 25.2 Hopcroft 的校验环技巧

课件给出一种更巧妙的设计：

- 用两个数组 `F[]` 和 `T[]`
- 再维护一个栈顶 `top`

有效位满足：

\[
T[F[k]] = k,\quad F[T[k]] = k
\]

### 25.3 好处

这样一来：

- `reset()` 只需把 `top = 0`
- 真正做到 **O(1)** 复位

这是“用额外结构避免整体清空”的经典技巧。

## 26. 本章主线总结

### 26.1 抽象与实现分离

- ADT 负责定义接口与语义；
- 数据结构负责给出具体实现与复杂度。

### 26.2 向量是“受管理的数组”

它保留了数组的顺序存储和随机访问优点，同时通过封装提供了更多统一接口。

### 26.3 倍增扩容体现分摊思想

- 固定增量扩容会导致平方级总代价；
- 倍增扩容使插入的分摊复杂度降为 `O(1)`。

### 26.4 有序性是极其重要的结构信息

一旦向量有序：

- 去重可从 `O(n^2)` 降到 `O(n)`
- 查找可从 `O(n)` 降到 `O(log n)`
- 插入位置也能更优雅地确定

### 26.5 不同查找算法对应不同假设

- 顺序查找：无序数据
- 二分查找：有序数据
- 插值查找：有序且近似均匀分布的数据

### 26.6 排序与位图体现“结构换效率”

- 归并排序用额外空间换稳定的 `O(n log n)` 时间
- 位图用值域空间换大规模去重和筛法的高效率

---

## 27. 一句话总结

> 向量并不只是“能自动扩容的数组”，而是顺序存储结构在抽象接口、动态空间管理、查找、排序、去重与位运算应用等多个层面上的系统化实现；这一章真正重要的，不只是会写 `Vector`，而是学会如何围绕“连续存储”这一特征设计高效算法。
